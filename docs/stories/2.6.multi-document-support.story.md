## Story

**As a** usuário,
**I want** enviar 3 ou mais documentos num único Plano de Ataque identificando cada fonte,
**so that** eu possa incluir setores adicionais (RH, Financeiro, Operações) além de Comercial e Marketing na análise — gerando um Plano mais completo sem precisar refazer o processo.

## Acceptance Criteria

1. Sistema aceita de 2 a 5 documentos por sessão de Plano de Ataque (limite já definido na Story 2.2 — formalizar comportamento guiado)
2. Cada documento é identificado por rótulo: extraído automaticamente do nome do arquivo ou atribuído pelo usuário após o upload
3. Seção dedicada gerada no Notion para cada documento: `heading_2` "📄 [Rótulo]" + análise individual (FR18)
4. Análise Integrada considera e menciona todos os documentos fornecidos (FR18)
5. Interface guia o usuário após cada documento recebido: `"📄 *Setor RH.pdf* recebido (3/5). Envie o próximo documento ou digite **pronto** para gerar o Plano de Ataque."`
6. Usuário pode rotular cada documento digitando o nome após o envio: `"chama de setor RH"` ou `"isso é o financeiro"` → rótulo atribuído ao último documento recebido
7. Ao digitar "pronto" com pelo menos 2 documentos → geração do plano iniciada
8. Ao digitar "pronto" com apenas 1 documento → aviso: `"⚠️ Adicione pelo menos 1 documento para gerar o Plano de Ataque. Envie mais arquivos ou digite **cancelar**."`

### `extractLabelFromFilename` — Implementação de Referência

```typescript
export function extractLabelFromFilename(filename: string): string {
  // Remove extensão
  const noExt = filename.replace(/\.[^/.]+$/, '').trim();
  if (!noExt) return '';

  // Remove padrões de data/ano: _2026, -2026, _jan_2026
  const noDate = noExt.replace(/[-_]\d{4}([-_]\d{2})?([-_]\d{2})?/g, '');

  // Converte separadores para espaço
  const spaced = noDate.replace(/[-_]+/g, ' ').trim();
  if (!spaced) return '';

  // Capitaliza cada palavra
  return spaced
    .split(' ')
    .map(word => word.charAt(0).toUpperCase() + word.slice(1).toLowerCase())
    .join(' ');
}

// Uso no DocumentService:
const label = extractLabelFromFilename(filename) || `Documento ${docIndex + 1}`;
```

### Fluxo de Estado no ChatService

```
Estado: IDLE (sem pendingDocuments)
  → Recebe documento → adiciona ao array → vai para COLLECTING

Estado: COLLECTING (1..4 pendingDocuments)
  → Recebe documento → adiciona + mensagem guia "envie mais ou pronto"
  → Recebe comando label → setDocumentLabel(-1, label) + confirma
  → Recebe "pronto" com count=1 → aviso "mínimo 2 documentos"
  → Recebe "pronto" com count≥2 → confirma lista + vai para ANALYZING
  → Recebe "cancelar" → clearPendingDocuments + mensagem cancelamento + IDLE

Estado: COLLECTING_FULL (5 pendingDocuments)
  → Recebe documento → rejeita com "limite atingido"
  → Recebe "pronto" → confirma lista + vai para ANALYZING

Estado: ANALYZING
  → Streaming da análise → preview enviado
  → Aguarda confirmação do usuário → CREATING_NOTION

Estado: CREATING_NOTION
  → Cria página Notion → auto-fill + histórico (Story 2.5) → IDLE
```

**Nota de performance (NFR1):**
Com múltiplos documentos, o tempo de análise pode ultrapassar 60s para 5 documentos grandes. Estratégia:
- Enviar streaming progressivo por documento (usuário vê cada seção sendo gerada)
- Tempo total aceitável: 60s × N_docs (definido em PRD como <60s por doc)
- Implementar timeout de 5 minutos por análise completa com fallback para análise parcial
