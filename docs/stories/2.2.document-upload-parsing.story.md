## Story

**As a** usuário,
**I want** enviar arquivos PDF ou Word via Telegram ou Web UI e ter o conteúdo extraído automaticamente,
**so that** eu possa iniciar o fluxo de geração do Plano de Ataque sem processar documentos manualmente.

## Acceptance Criteria

1. Recebimento de arquivos PDF (`.pdf`) e Word (`.doc`, `.docx`) via **Telegram Bot** e **Web UI**
2. Extração de texto de PDFs com `pdf-parse` — retorna texto bruto do documento
3. Extração de texto de Word/DOCX com `mammoth` — retorna texto bruto sem formatação HTML
4. Arquivo processado **em memória** (Buffer) e nunca salvo em disco — NFR11
5. Suporte a múltiplos arquivos em sequência: até **5 documentos** por sessão de Plano de Ataque
6. Feedback imediato ao usuário após cada upload: `"📄 *Setor Comercial.pdf* recebido e processado (3.2KB)"`
7. Arquivos acima de **10MB** rejeitados antes do parsing com mensagem clara: `"❌ Arquivo muito grande (12MB). Limite: 10MB."`
8. Endpoint `POST /api/documents/upload` recebe multipart/form-data, retorna `ProcessedDocument`
9. Handler Telegram para `message:document` — baixa arquivo via Bot API, processa em memória, retorna feedback
10. Tipos de arquivo não suportados rejeitados com código `DOC_001` e mensagem amigável

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Malformed PDF crashes parser** | Service error | Low | Wrap with try-catch; log detailed errors |
| **Word document with complex formatting** | Text extraction loses structure | Medium | Test with real client Word documents |
| **File size limit bypass** | Denial of service | Low | Validate size before stream processing |
| **Memory leak on large files** | Server memory grows | Low | Process in chunks; clear buffer after extraction |

## Out of Scope

- ❌ **Image files (JPG, PNG)** — Text documents only
- ❌ **OCR for scanned PDFs** — Assume text-based PDFs
- ❌ **Batch upload** — Single file per request (Story 2.6 for multi)
- ❌ **File persistence** — Memory only

### PdfParser — Implementação de Referência

```typescript
// packages/core/src/documents/PdfParser.ts
import pdfParse from 'pdf-parse';
import { AppError } from '../errors/AppError';
import type { ProcessedDocument } from '@aria/shared';

export class PdfParser {
  async parse(buffer: Buffer, filename: string): Promise<ProcessedDocument> {
    try {
      const result = await pdfParse(buffer);
      return {
        id: crypto.randomUUID(),
        originalName: filename,
        mimeType: 'application/pdf',
        extractedText: result.text.trim(),
        uploadedAt: new Date(),
        processedAt: new Date(),
      };
    } catch (error) {
      throw new AppError(
        `Failed to parse PDF: ${filename}`,
        'DOC_003',
        { statusCode: 422, cause: error }
      );
    }
  }
}
```

### DocumentService — Sessão In-Memory

```typescript
// packages/core/src/documents/DocumentService.ts
const MAX_DOCS_PER_SESSION = 5;
const MAX_FILE_SIZE = 10 * 1024 * 1024; // 10MB

export class DocumentService {
  private sessionDocs = new Map<string, ProcessedDocument[]>();

  async processDocument(
    buffer: Buffer,
    filename: string,
    mimeType: string
  ): Promise<ProcessedDocument> {
    // Validação de tamanho
    if (buffer.length > MAX_FILE_SIZE) {
      const sizeMb = (buffer.length / 1024 / 1024).toFixed(1);
      throw new AppError(
        `File too large: ${sizeMb}MB. Limit: 10MB`,
        'DOC_002',
        { statusCode: 413 }
      );
    }

    // Validação de tipo
    if (!SUPPORTED_MIME_TYPES.includes(mimeType as SupportedMimeType)) {
      throw new AppError(
        `Unsupported file type: ${mimeType}`,
        'DOC_001',
        { statusCode: 415 }
      );
    }

    // Dispatch para parser correto
    if (mimeType === 'application/pdf') {
      return new PdfParser().parse(buffer, filename);
    }
    return new DocxParser().parse(buffer, filename);
  }

  addToSession(sessionId: string, doc: ProcessedDocument): void {
    const docs = this.sessionDocs.get(sessionId) ?? [];
    if (docs.length >= MAX_DOCS_PER_SESSION) {
      throw new AppError('Maximum 5 documents per session', 'DOC_004', { statusCode: 400 });
    }
    this.sessionDocs.set(sessionId, [...docs, doc]);
  }

  getSessionDocuments(sessionId: string): ProcessedDocument[] {
    return this.sessionDocs.get(sessionId) ?? [];
  }

  clearSession(sessionId: string): void {
    this.sessionDocs.delete(sessionId);
  }
}

export const documentService = new DocumentService(); // Singleton
```

### Telegram Document Handler — Referência

```typescript
// apps/bot/src/handlers/document.handler.ts
import { Context } from 'grammy';
import { documentService } from '@aria/core';
import { SUPPORTED_MIME_TYPES } from '@aria/shared';

export async function documentHandler(ctx: Context) {
  const doc = ctx.message?.document;
  if (!doc) return;

  const sessionId = ctx.session.sessionId;
  const fileName = doc.file_name ?? 'documento';
  const mimeType = doc.mime_type ?? '';
  const fileSize = doc.file_size ?? 0;

  // Log sem conteúdo (NFR13)
  // logger.info({ sessionId, fileName, fileSize }, 'Document received');

  // Validações antes do download
  if (fileSize > 10 * 1024 * 1024) {
    const sizeMb = (fileSize / 1024 / 1024).toFixed(1);
    await ctx.reply(`❌ Arquivo muito grande (${sizeMb}MB)\\. Limite: 10MB\\.`);
    return;
  }

  if (!SUPPORTED_MIME_TYPES.includes(mimeType as any)) {
    await ctx.reply('❌ Formato não suportado\\. Envie arquivos PDF ou Word \\(\\.docx\\)\\.');
    return;
  }

  // Download do arquivo via Bot API
  await ctx.reply('⏳ Processando documento\\.\\.\\.');
  const file = await ctx.api.getFile(doc.file_id);
  const fileUrl = `https://api.telegram.org/file/bot${process.env.TELEGRAM_BOT_TOKEN}/${file.file_path}`;
  const response = await fetch(fileUrl);
  const buffer = Buffer.from(await response.arrayBuffer());

  // Parse em memória (NFR11)
  const processed = await documentService.processDocument(buffer, fileName, mimeType);
  documentService.addToSession(sessionId, processed);

  const sizeKb = (fileSize / 1024).toFixed(1);
  const count = documentService.getSessionDocuments(sessionId).length;

  await ctx.reply(
    `📄 *${fileName}* recebido e processado \\(${sizeKb}KB\\)\n\n` +
    `Total nesta sessão: ${count}/5\n\n` +
    `Envie mais documentos ou diga *pronto* para gerar o Plano de Ataque\\.`,
    { parse_mode: 'MarkdownV2' }
  );
}
```

### Validação de MIME Types

```
PDF:  'application/pdf'
DOC:  'application/msword'
DOCX: 'application/vnd.openxmlformats-officedocument.wordprocessingml.document'
```

**Nota:** Telegram pode enviar `mime_type` como `application/octet-stream` para arquivos sem tipo claro. Considerar usar extensão do arquivo (`file_name`) como fallback para detectar tipo.

### NFR11 — Garantia de Não-Persistência

CRÍTICO: Em nenhum momento do fluxo o arquivo deve ser escrito em disco.
- `pdf-parse`: aceita `Buffer` diretamente ✅
- `mammoth`: aceita `{ buffer: Buffer }` ✅
- `@fastify/multipart`: usar `.toBuffer()` ao invés de `.pipe(fs.createWriteStream(...))` ✅
- Telegram: `Buffer.from(await response.arrayBuffer())` — nunca usar `fs.writeFile` ✅
