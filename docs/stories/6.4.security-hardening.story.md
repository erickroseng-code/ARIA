# Story 6.4: Security Hardening & Compliance

## Status
Draft

## Story

**As a** security officer,
**I want** implementar práticas de segurança em todas as camadas,
**so that** os dados dos usuários sejam protegidos contra ataques.

## Acceptance Criteria

1. Authentication: JWT tokens (short-lived) + refresh tokens (long-lived)
2. Password: bcrypt hashing + salting; minimum 8 chars
3. Rate limiting: per-endpoint limits (10 req/sec for auth, 100 req/sec for API)
4. Input validation: Zod schemas for all user inputs (prevent injection)
5. CORS: whitelist production domain only (block cross-origin)
6. HTTPS: TLS 1.3 enforced (automatic via deployment platform)
7. Headers: CSP, X-Frame-Options, X-Content-Type-Options set
8. Secrets: never commit to git; use environment variables only
9. Dependencies: security audits (npm audit); update regularly
10. Logging: audit logs for sensitive operations (login, data access)
