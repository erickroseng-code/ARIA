## Story

**As a** usuário,
**I want** dizer "cliente: Empresa X" e a ARIA localizar automaticamente o perfil dessa empresa no Notion,
**so that** eu possa iniciar fluxos de trabalho (Plano de Ataque, tarefas, status) sem precisar copiar IDs ou navegar manualmente.

## Acceptance Criteria

1. Integração com Notion API configurada via `NOTION_API_KEY` e `NOTION_DATABASE_ID` no `.env`
2. Busca de página de cliente por nome — parcial e insensível a maiúsculas (ex: "empresa" encontra "Empresa X Ltda")
3. Quando cliente encontrado: ARIA retorna confirmação `"✅ Cliente *Empresa X* encontrado"` e armazena `notionPageId` no `activeClientId` da sessão
4. Quando não encontrado: ARIA retorna lista dos **3 clientes mais similares** para o usuário escolher (por similaridade de nome)
5. `activeClientId` atualizado no `ContextStore` após confirmação do cliente
6. Leitura de propriedades básicas do perfil: nome, segmento, responsável — retornadas como `ClientRef`
7. Cache in-memory de 5 minutos para listagem de clientes (evitar chamadas repetidas à Notion API)
8. Endpoint `GET /api/clients` retorna lista de `ClientRef[]` com cache
9. Endpoint `GET /api/clients/:id` retorna `ClientProfile` completo do cliente
10. Tratamento de erros Notion API com código `NOTION_001` e mensagem amigável em português

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Notion API rate limit** | Requests throttled | Low | Implement 5-min cache; log rate limit hits |
| **Fuzzy matching returns wrong client** | User selects incorrect client | Medium | Always show top 3 matches; require confirmation |
| **Cache stale** | User sees deleted client | Low | 5-min TTL acceptable for MVP |
| **Empty database** | No clients to select | Low | Show helpful error message |

### NotionClient — Implementação de Referência

```typescript
// packages/integrations/src/notion/notion.client.ts
import { Client } from '@notionhq/client';
import type { ClientRef, ClientProfile } from '@aria/shared';

interface CacheEntry<T> {
  data: T;
  expiresAt: number;
}

export class NotionClient {
  private client: Client;
  private cache = new Map<string, CacheEntry<ClientRef[]>>();
  private readonly CACHE_TTL = 5 * 60 * 1000; // 5 minutos

  constructor(apiKey: string, private databaseId: string) {
    this.client = new Client({ auth: apiKey });
  }

  async listClients(): Promise<ClientRef[]> {
    const cached = this.cache.get('all_clients');
    if (cached && cached.expiresAt > Date.now()) {
      return cached.data;
    }

    const response = await this.client.databases.query({
      database_id: this.databaseId,
      page_size: 100,
    });

    const clients = response.results.map((page: any) => ({
      id: page.id,
      name: page.properties?.Name?.title?.[0]?.plain_text ?? 'Sem nome',
      notionPageId: page.id,
    }));

    this.cache.set('all_clients', { data: clients, expiresAt: Date.now() + this.CACHE_TTL });
    return clients;
  }

  async getTopMatches(query: string, limit = 3): Promise<ClientRef[]> {
    const all = await this.listClients();
    const q = query.toLowerCase();

    // Scoring: exact match = 3, starts with = 2, includes = 1
    const scored = all
      .map((c) => {
        const name = c.name.toLowerCase();
        const score = name === q ? 3 : name.startsWith(q) ? 2 : name.includes(q) ? 1 : 0;
        return { client: c, score };
      })
      .filter((x) => x.score > 0)
      .sort((a, b) => b.score - a.score);

    return scored.slice(0, limit).map((x) => x.client);
  }

  async getClientProfile(pageId: string): Promise<ClientProfile> {
    const page = await this.client.pages.retrieve({ page_id: pageId }) as any;
    const props = page.properties;

    return {
      id: page.id,
      name: props?.Name?.title?.[0]?.plain_text ?? '',
      notionPageId: page.id,
      company: props?.Company?.rich_text?.[0]?.plain_text,
      email: props?.Email?.email ?? undefined,
      tags: props?.Tags?.multi_select?.map((t: any) => t.name) ?? [],
      recentTasks: [],
      createdAt: new Date(page.created_time),
      updatedAt: new Date(page.last_edited_time),
    };
  }

  invalidateCache(): void {
    this.cache.clear();
  }
}
```

### ClientService — Fluxo de Lookup

```typescript
// packages/core/src/clients/ClientService.ts
export class ClientService {
  constructor(
    private notionClient: NotionClient,
    private contextStore: ContextStore,
  ) {}

  async handleClientLookup(
    clientName: string,
    sessionId: string,
    userId: string
  ): Promise<string> {
    try {
      const matches = await this.notionClient.getTopMatches(clientName, 3);

      if (matches.length === 0) {
        return `❌ Não encontrei nenhum cliente com o nome "${clientName}" no Notion.\n\nVerifique o nome ou acesse o Notion diretamente.`;
      }

      if (matches.length === 1 || matches[0].name.toLowerCase() === clientName.toLowerCase()) {
        const client = matches[0];
        this.contextStore.setActiveClient(userId, client.notionPageId);
        return `✅ Cliente *${client.name}* encontrado e definido como ativo.`;
      }

      // Múltiplos matches — pedir confirmação
      const list = matches
        .map((c, i) => `${i + 1}. ${c.name}`)
        .join('\n');
      return `Encontrei ${matches.length} clientes similares:\n\n${list}\n\nDigite o número para selecionar.`;
    } catch (error) {
      throw new AppError(
        'Erro ao buscar cliente no Notion',
        'NOTION_001',
        { statusCode: 502, cause: error }
      );
    }
  }
}
```

### Mapeamento de Propriedades Notion

A estrutura exata das propriedades depende do database Notion do usuário. O `NotionClient` deve ser **defensivo** ao ler propriedades — usar optional chaining (`?.`) em todos os acessos.

**Propriedades esperadas no database de clientes:**
| Propriedade Notion | Tipo | Campo `ClientRef`/`ClientProfile` |
|-------------------|------|----------------------------------|
| `Name` | title | `name` |
| `Company` | rich_text | `company` |
| `Email` | email | `email` |
| `Phone` | phone_number | `phone` |
| `Tags` | multi_select | `tags` |
| `Segment` | select | campo futuro |

**Nota:** Se o database do usuário tiver nomes de propriedades diferentes, o `notion.helpers.ts` deve ter uma função de mapeamento configurável.
