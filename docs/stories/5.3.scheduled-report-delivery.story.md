# Story 5.3: Scheduled Report Delivery & Distribution

## Status
Done

## Story

**As a** usuário,
**I want** agendar relatórios para serem entregues automaticamente,
**so that** eu receba insights regularmente sem precisar solicitar.

## Acceptance Criteria

1. Schedule configuration: frequency (daily, weekly, monthly, custom day/time)
2. User specifies delivery channels: Telegram, email (Phase 2), Notion append
3. Scheduled report generation: Bull queue + cron jobs
4. Report delivery: exactly on scheduled time (±5 min tolerance)
5. Timezone aware: schedule respects user's timezone
6. Failed deliveries: queue retry (3x with exponential backoff)
7. Delivery history: stored in database (user can view past deliveries)
8. User can pause/resume scheduling
9. User can modify schedule at any time
10. Missed reports tracked: show on dashboard when user logs in
