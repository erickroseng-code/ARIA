# Story 12.1: Weekly Batch Endpoint

## Status
Done

## Story

**As a** sistema de automação (GitHub Actions),
**I want** chamar um único endpoint que gera carrosséis completos para múltiplos tópicos,
**so that** toda segunda-feira o pipeline gere automaticamente o conteúdo da semana sem intervenção manual.

## Acceptance Criteria

1. **`POST /api/maverick/weekly-batch`** aceita body:
   ```json
   {
     "topics": ["emagrecimento feminino", "mindset financeiro"],
     "theme": "dark",
     "scriptOptions": { "maxAgeDays": 45 }
   }
   ```
2. **Para cada tópico**, o endpoint executa:
   - Gera keywords via LLM (baseado no tópico como username genérico)
   - Gera scripts via Maverick copywriter pipeline
   - Para o melhor script, gera `CarouselStructure` via `generateCarouselStructure`
   - Gera HTML via `generateCarouselHtml`
3. **Response** (síncrono, aguarda todos os tópicos):
   ```json
   {
     "batchId": "batch-20260310-abc123",
     "generatedAt": "2026-03-10T11:00:00.000Z",
     "topics": [
       {
         "topic": "emagrecimento feminino",
         "status": "success",
         "carousel": {
           "title": "...",
           "total_slides": 6,
           "slides": [...],
           "htmlExport": "<html>..."
         }
       }
     ]
   }
   ```
4. **Tópico com falha** não quebra o batch: `status: "error", error: "mensagem"`.
5. **Máximo de 10 tópicos** por request (retorna 400 se exceder).
6. **Timeout de 5 minutos** por tópico (retorna `status: "timeout"` se exceder).
7. **`batchId` salvo** em memória/arquivo temporário para que a Story 12.3 possa referenciar o batch.

## Tasks / Subtasks

- [x] **Task 1** — Criar `weekly-batch.service.ts` (AC: 2, 4, 6)
  ```ts
  // aria/apps/api/src/modules/maverick/weekly-batch.service.ts

  import { generateCarouselStructure, ScriptInput } from './carousel-designer/index';
  import { generateCarouselHtml } from './carousel-designer/html-export';
  import { randomBytes } from 'crypto';

  export interface BatchTopic {
    topic: string;
    status: 'success' | 'error' | 'timeout';
    carousel?: {
      title: string;
      total_slides: number;
      slides: any[];
      htmlExport: string;
    };
    error?: string;
  }

  export interface BatchResult {
    batchId: string;
    generatedAt: string;
    topics: BatchTopic[];
  }

  // In-memory store for batch results (cleared on restart)
  const batchStore = new Map<string, BatchResult>();

  export async function runWeeklyBatch(
    topics: string[],
    theme: 'dark' | 'light' = 'dark',
    openrouterApiKey: string,
    scriptOptions: { maxAgeDays?: number } = {}
  ): Promise<BatchResult> {
    const batchId = `batch-${new Date().toISOString().slice(0, 10)}-${randomBytes(4).toString('hex')}`;
    const results: BatchTopic[] = [];

    for (const topic of topics) {
      try {
        const result = await processTopic(topic, theme, openrouterApiKey, scriptOptions);
        results.push(result);
      } catch (err: any) {
        results.push({ topic, status: 'error', error: err.message });
      }
    }

    const batch: BatchResult = {
      batchId,
      generatedAt: new Date().toISOString(),
      topics: results,
    };

    batchStore.set(batchId, batch);
    // Auto-cleanup após 24h
    setTimeout(() => batchStore.delete(batchId), 24 * 60 * 60 * 1000);

    return batch;
  }

  export function getBatch(batchId: string): BatchResult | undefined {
    return batchStore.get(batchId);
  }

  async function processTopic(
    topic: string,
    theme: 'dark' | 'light',
    openrouterApiKey: string,
    scriptOptions: { maxAgeDays?: number }
  ): Promise<BatchTopic> {
    // 1. Gerar keywords via LLM
    const keywords = await generateKeywords(topic, openrouterApiKey);

    // 2. Gerar script simples baseado no tópico (sem scraping de Instagram)
    const script = await generateSimpleScript(topic, keywords, openrouterApiKey);

    // 3. Gerar estrutura de carousel
    const carousel = generateCarouselStructure(script);

    // 4. Gerar HTML
    const htmlExport = generateCarouselHtml(carousel, theme);

    return {
      topic,
      status: 'success',
      carousel: {
        title: carousel.title,
        total_slides: carousel.total_slides,
        slides: carousel.slides,
        htmlExport,
      },
    };
  }
  ```

    if (!Array.isArray(topics) || topics.length === 0) {

## Dev Notes

### Geração de Script sem Scraping
Para o batch automático, não é viável esperar o scraping completo do Instagram por tópico. O `generateSimpleScript` usa o LLM diretamente com o tópico como input, gerando um script de carrossel educativo baseado no conhecimento do modelo.

Prompt base para `generateSimpleScript`:
```
Crie um script de carrossel do Instagram sobre: "{topic}"
Formato JSON:
{
  "title": "título chamativo",
  "format": "Carrossel Educativo",
  "hook": "primeira frase impactante que vai na capa",
  "body": "corpo do conteúdo dividido em 3-4 parágrafos curtos separados por linha em branco",
  "visual_cues": ["dica visual slide 1", "dica visual slide 2", ...],
  "cta": "chamada para ação final"
}
```

### batchStore
Para MVP, o `batchStore` é in-memory. Em produção (Render), isso funciona enquanto o serviço está ativo — mas é perdido no restart. A Story 12.3 precisará do batchId para servir o download.
