# Story 13.1: Maverick — Script Intelligence Pipeline (Brief Generator + Viral Validator)

## Status
Ready for Review

## Story

**As a** criador de conteúdo usando o Maverick,
**I want** que o sistema sintetize os dados da análise de perfil em um brief criativo antes de gerar o roteiro,
**so that** o roteiro gerado seja baseado em dores reais do público e padrões comprovados de gancho viral — e não em instruções genéricas.

## Acceptance Criteria

1. **`BriefGenerator` service** implementado em `squads/maverick/apps/api/src/services/brief-generator/index.ts` com método `generate(input: BriefInput): Promise<string>` que retorna um brief narrativo de 100–150 palavras.

2. **`PATTERN_LIBRARY`** — array de 9 padrões de gancho extraídos dos scripts Penoni, com campos `id`, `name`, `formula`, `whyItWorks`, `example` — exportado como constante do módulo BriefGenerator.

3. **`matchPattern(topic: string, analysisData: AnalysisData): HookPattern`** — função que seleciona o padrão mais adequado de forma programática (sem LLM), usando keyword matching nos campos `pilares_conteudo`, `pontos_melhoria`, `content_tilt` da análise.

4. **`ViralValidator`** — função `validate(script: string): ValidationResult` implementada em `squads/maverick/apps/api/src/services/brief-generator/viral-validator.ts` com as checagens:
   - Hook ≤ 15 palavras (primeira linha do `[GANCHO]`)
   - Presença de número/estatística no script (`/\d+/`)
   - Presença das 3 seções: `[GANCHO]`, `[DESENVOLVIMENTO]`, `[CTA]`
   - Contagem de palavras entre 220 e 400
   - Ausência de palavras banidas: `transformação`, `jornada`, `incrível`, `poderoso`, `revolucionário`, `guru`
   - Retorna `{ score: number (0-5), issues: string[], passed: boolean }` (passed = score ≥ 3)

5. **Integração no endpoint `/api/generate-script`** — Stage 0 inserido ANTES do Pass 1 atual:
   - Chamar `matchPattern()` para selecionar padrão
   - Chamar Scholar RAG com 1 chunk focado (query: `gancho ${topic} ${patternName}`)
   - Chamar `briefGenerator.generate()` com os dados da análise + padrão selecionado
   - O brief gerado substitui o `trendBlock` dentro do Pass 1 prompt

6. **Integração da `ViralValidator`** — após o Pass 2 atual:
   - Chamar `validate(finalScript)`
   - Se `passed === false` E `score < 3`: re-gerar Pass 2 com prompt de correção (`fixPrompt`) que lista os `issues` específicos
   - Máximo 1 re-tentativa de correção (evita loop infinito)
   - Logar resultado: `[ViralValidator] Score: X/5 — issues: [...]`

7. **`AnalysisData` interface** definida e usada no endpoint — campos extraídos do body quando `analysisContext` for passado:
   - `pilares_conteudo: string[]`
   - `pontos_melhoria: string[]`
   - `content_tilt?: string`
   - `top_posts?: Array<{ description: string }>`

8. **Meta do response** atualizado com: `briefGenerated: boolean`, `hookPattern: string`, `viralScore: number`, `validationIssues: string[]`.

  export function matchPattern(topic: string, analysisData?: AnalysisData): HookPattern {
    const searchText = [
      topic,
      ...(analysisData?.pilares_conteudo || []),
      ...(analysisData?.pontos_melhoria || []),
      analysisData?.content_tilt || '',
      ...(analysisData?.top_posts?.map(p => p.description) || []),
    ].join(' ').toLowerCase();

    let bestPattern = PATTERN_LIBRARY[0];
    let bestScore = 0;

    for (const pattern of PATTERN_LIBRARY) {
      const score = pattern.keywords.filter(kw => searchText.includes(kw)).length;
      if (score > bestScore) {
        bestScore = score;
        bestPattern = pattern;
      }
    }

    // Default: resultado_numero é o padrão mais versátil se não houver match
    if (bestScore === 0) {
      return PATTERN_LIBRARY.find(p => p.id === 'resultado_numero') || PATTERN_LIBRARY[0];
    }

    return bestPattern;
  }

- [x] **Task 3** — Criar `ViralValidator` em `brief-generator/viral-validator.ts` (AC: 4)

  ```typescript
  // squads/maverick/apps/api/src/services/brief-generator/viral-validator.ts

  export interface ValidationResult {
    score: number;     // 0-5
    issues: string[];
    passed: boolean;   // score >= 3
  }

  const BANNED_WORDS = ['transformação', 'jornada', 'incrível', 'poderoso', 'revolucionário', 'guru'];

  export function validate(script: string): ValidationResult {
    const issues: string[] = [];
    let score = 5;

    // 1. Hook ≤ 15 palavras
    const ganchoMatch = script.match(/\[GANCHO\]\s*\n(.*)/i);
    if (ganchoMatch) {
      const hookLine = ganchoMatch[1].trim();
      const wordCount = hookLine.split(/\s+/).length;
      if (wordCount > 15) {
        issues.push(`Gancho muito longo: ${wordCount} palavras (máx 15)`);
        score--;
      }
    } else {
      issues.push('Seção [GANCHO] não encontrada');
      score--;
    }

    // 2. Presença de número/estatística
    if (!/\d+/.test(script)) {
      issues.push('Sem números ou estatísticas — adicione especificidade');
      score--;
    }

    // 3. Seções obrigatórias
    const hasDesenvolvimento = /\[DESENVOLVIMENTO\]/i.test(script);
    const hasCta = /\[CTA\]/i.test(script);
    if (!hasDesenvolvimento || !hasCta) {
      issues.push('Seções [DESENVOLVIMENTO] e/ou [CTA] ausentes');
      score--;
    }

    // 4. Contagem de palavras (220–400)
    const wordCount = script.split(/\s+/).filter(w => w.length > 0).length;
    if (wordCount < 220) {
      issues.push(`Roteiro curto demais: ${wordCount} palavras (mín 220)`);
      score--;
    } else if (wordCount > 400) {
      issues.push(`Roteiro longo demais: ${wordCount} palavras (máx 400)`);
      score--;
    }

    // 5. Palavras banidas
    const foundBanned = BANNED_WORDS.filter(w => script.toLowerCase().includes(w));
    if (foundBanned.length > 0) {
      issues.push(`Palavras de guru encontradas: ${foundBanned.join(', ')}`);
      score--;
    }

    return {
      score: Math.max(0, score),
      issues,
      passed: Math.max(0, score) >= 3,
    };
  }
  ```

### Nota sobre os modelos free

Os modelos free do OpenRouter têm limite de contexto. O brief gerado (~150 palavras) substitui o `trendBlock` de forma mais focada, mantendo o prompt do Pass 1 em ~600 tokens — dentro dos limites dos modelos.

### Padrão de fallback

Se `BriefGenerator.generate()` falhar (timeout, modelo offline), o código usa `trendBlock` como fallback — o comportamento anterior é preservado. Nunca quebrar o endpoint por falha no Brief Generator.
