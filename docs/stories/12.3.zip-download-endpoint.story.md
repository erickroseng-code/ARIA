# Story 12.3: ZIP + Download Endpoint

## Status
Done

## Story

**As a** criador de conteúdo ou sistema de automação,
**I want** baixar todos os slides do batch em um único arquivo ZIP,
**so that** possa publicar os carrosséis no Instagram sem precisar baixar imagem por imagem.

## Acceptance Criteria

1. **`GET /api/maverick/weekly-batch/:batchId/download`** retorna o ZIP como download.
2. **Estrutura do ZIP:**
   ```
   carrossets-{DATA}/
   ├── topico-01-emagrecimento-feminino/
   │   ├── slide-01.png
   │   ├── slide-02.png
   │   └── slide-06.png
   └── topico-02-mindset-financeiro/
       ├── slide-01.png
       └── slide-05.png
   ```
3. **Headers corretos:** `Content-Type: application/zip`, `Content-Disposition: attachment; filename="carrossets-{DATA}.zip"`.
4. **404** se `batchId` não encontrado ou expirado.
5. **Fallback HTML:** se o batch não tem screenshots (Playwright indisponível), o ZIP contém arquivos `.html` ao invés de `.png`:
   ```
   carrossets-{DATA}/
   ├── topico-01-emagrecimento-feminino/
   │   ├── slide-01.html
   │   └── carousel-completo.html
   ```
6. **Telegram upload:** endpoint alternativo `POST /api/maverick/weekly-batch/:batchId/send-telegram` envia o ZIP diretamente para o chat Telegram configurado (arquivo até 50MB).
7. **Cleanup:** após download ou 24h (o que ocorrer primeiro), arquivos temporários são deletados.

## Scope

**IN:**
- Criar endpoint `GET /api/maverick/weekly-batch/:batchId/download`
- Criar endpoint `POST /api/maverick/weekly-batch/:batchId/send-telegram`
- Usar `archiver` ou `adm-zip` para criar ZIP em memória
- Adicionar lógica de cleanup pós-download

**OUT:**
- Não fazer upload para S3 ou storage externo
- Não criar URL pública separada (a URL do endpoint Render já é pública)
- Não criar interface web para o download (apenas o link direto)

- [x] **Task 2** — Criar `zip.service.ts` (AC: 1, 2, 3, 5)
  ```ts
  // aria/apps/api/src/modules/maverick/zip.service.ts

  import * as fs from 'fs';
  import * as path from 'path';
  import archiver from 'archiver';
  import { BatchResult } from './weekly-batch.service';

  export function createBatchZip(batch: BatchResult): Promise<Buffer> {
    return new Promise((resolve, reject) => {
      const chunks: Buffer[] = [];
      const archive = archiver('zip', { zlib: { level: 6 } });

      archive.on('data', (chunk: Buffer) => chunks.push(chunk));
      archive.on('end', () => resolve(Buffer.concat(chunks)));
      archive.on('error', reject);

      const date = batch.generatedAt.slice(0, 10);
      const folderName = `carrossets-${date}`;

      batch.topics.forEach((topic, i) => {
        if (topic.status !== 'success') return;

        const slug = topic.topic
          .toLowerCase()
          .replace(/\s+/g, '-')
          .replace(/[^a-z0-9-]/g, '')
          .slice(0, 40);
        const topicFolder = `${folderName}/topico-${String(i + 1).padStart(2, '0')}-${slug}`;

        // PNGs se disponíveis
        const batchTmpDir = `/tmp/${batch.batchId}/topico-${i + 1}`;
        if (fs.existsSync(batchTmpDir)) {
          const pngs = fs.readdirSync(batchTmpDir).filter(f => f.endsWith('.png'));
          pngs.forEach(png => {
            archive.file(path.join(batchTmpDir, png), { name: `${topicFolder}/${png}` });
          });
        } else if (topic.carousel?.htmlExport) {
          // Fallback: HTML
          archive.append(topic.carousel.htmlExport, { name: `${topicFolder}/carousel-completo.html` });
        }
      });

      archive.finalize();
    });
  }
  ```

- [x] **Task 3** — Adicionar endpoints em `maverick.routes.ts` (AC: 1, 3, 4, 6, 7)
  ```ts
  // GET /api/maverick/weekly-batch/:batchId/download
  fastify.get('/weekly-batch/:batchId/download', async (req, reply) => {
    const { batchId } = req.params as { batchId: string };
    const batch = getBatch(batchId);
    if (!batch) return reply.status(404).send({ error: 'Batch não encontrado ou expirado' });

    const zipBuffer = await createBatchZip(batch);
    const date = batch.generatedAt.slice(0, 10);
    const filename = `carrossets-${date}.zip`;

    return reply
      .header('Content-Type', 'application/zip')
      .header('Content-Disposition', `attachment; filename="${filename}"`)
      .send(zipBuffer);
  });

  // POST /api/maverick/weekly-batch/:batchId/send-telegram
  fastify.post('/weekly-batch/:batchId/send-telegram', async (req, reply) => {
    const { batchId } = req.params as { batchId: string };
    const batch = getBatch(batchId);
    if (!batch) return reply.status(404).send({ error: 'Batch não encontrado ou expirado' });

    const token = process.env.TELEGRAM_BOT_TOKEN;
    const chatId = process.env.TELEGRAM_CHAT_ID;
    if (!token || !chatId) {
      return reply.status(500).send({ error: 'Telegram não configurado' });
    }

    const zipBuffer = await createBatchZip(batch);
    const date = batch.generatedAt.slice(0, 10);
    const filename = `carrossets-${date}.zip`;

    // Upload via Telegram sendDocument
    const formData = new FormData();
    formData.append('chat_id', chatId);
    formData.append('caption', `🎨 *Maverick Weekly* — Carrosséis da semana!\n📅 ${date}\n📦 ${batch.topics.filter(t => t.status === 'success').length} tópico(s) gerado(s)`);
    formData.append('parse_mode', 'Markdown');
    formData.append('document', new Blob([zipBuffer], { type: 'application/zip' }), filename);

    const res = await fetch(`https://api.telegram.org/bot${token}/sendDocument`, {
      method: 'POST',
      body: formData,
    });

    if (!res.ok) {
      const err = await res.text();
      return reply.status(500).send({ error: `Telegram error: ${err}` });
    }

    return reply.send({ success: true, filename });
  });
  ```

## Dev Notes

### archiver vs adm-zip
- `archiver`: API de streaming, melhor para arquivos grandes, mais popular
- `adm-zip`: API síncrona, mais simples para arquivos pequenos
- Recomendação: `archiver` para futuras extensões

### Tamanho estimado do ZIP
- 6 slides × 1080×1080 PNG ≈ 500KB-2MB por slide
- 3 tópicos × 6 slides = 18 PNGs ≈ 10-36MB por batch
- Bem abaixo do limite Telegram de 50MB para a maioria dos casos

### URL pública para download
No Render, o endpoint `GET /api/maverick/weekly-batch/:batchId/download` é acessível via `https://aria-api.onrender.com/api/maverick/weekly-batch/{batchId}/download`. Esta URL pode ser enviada no Telegram para download manual, ou o endpoint `send-telegram` pode enviar o arquivo diretamente.
