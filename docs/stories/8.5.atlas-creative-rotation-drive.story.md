# Story 8.5: Atlas — Rotação de Criativos via Google Drive + Aprovação Telegram

## Status
Ready for Dev

## Story

**As a** operador da Aria,
**I want** que o Atlas, ao detectar fadiga de criativo ou underperformance, acesse automaticamente a pasta de criativos para teste no Google Drive, selecione o novo criativo e proponha a troca (com nova copy) via Telegram para minha aprovação,
**so that** eu possa validar e aplicar novas variações de criativo com zero esforço manual, mantendo meu controle sobre as decisões.

## Acceptance Criteria

1. **Gatilho de Criativo:** Ao detectar qualquer dos critérios abaixo, o Atlas inicia o fluxo de rotação:
   - CTR < 1,5% (piso aceitável atingido)
   - CTR caiu > 30% vs. média histórica do próprio anúncio
   - Gasto > 3× CAC alvo sem conversão (stop-loss)
   - Hook Rate (3s) < 20%

2. **Acesso ao Google Drive:** O Atlas acessa a pasta configurada em `ATLAS_CREATIVE_DRIVE_FOLDER_ID` e lista os criativos disponíveis (imagens e vídeos).

3. **Seleção de Criativo:** O Atlas seleciona o criativo mais adequado (o que ainda não foi testado ou o menos recente).

4. **Geração de Copy:** O Atlas deve decidir entre:
   - **Criar nova copy:** novos título, descrição e texto principal baseados no histórico de performance e no produto
   - **Adaptar copy existente:** ajustar apenas o elemento problemático (ex: hook), mantendo o restante

5. **Proposta via Telegram:** Antes de qualquer alteração, enviar mensagem de proposta completa no formato:
   ```
   🎨 *Atlas — Proposta de Rotação de Criativo*
   📅 [data e hora]

   📊 Diagnóstico:
   Anúncio "[nome]" (ID: [id])
   • CTR: 0.8% (piso: 1.5%) ❌
   • Causa: Criativo com baixo engajamento inicial

   🆕 Novo criativo selecionado:
   📁 arquivo: [nome_do_arquivo.mp4/jpg]
   🔗 Drive: [link]

   ✍️ Copy proposta:
   Título: "[novo título]"
   Descrição: "[nova descrição]"
   Texto principal: "[novo texto]"

   ⚙️ Ação: Pausar "Ad_V2" → Subir novo anúncio com criativo acima

   Responda /aprovar_criativo_[hash] ou /reprovar_criativo_[hash]
   ```

6. **Execução pós-aprovação:** Somente após `/aprovar_criativo_[hash]`, o Atlas:
   - Pausa o anúncio com fadiga
   - Faz upload do criativo do Drive para a Meta
   - Cria um novo anúncio no mesmo conjunto com o criativo e copy aprovados
   - Envia confirmação via Telegram

7. **Isolamento de variável:** Ao criar o novo anúncio, manter o mesmo conjunto (adset) para garantir teste justo. Documentar claramente o que mudou.

8. **Regra de amostragem:** Aguardar mínimo de 800 impressões e 48h antes de avaliar o novo criativo.

## Scope

**IN:**
- Novo serviço `atlas-creative-service.ts` com funções:
  - `listDriveCreatives(folderId: string)` — lista criativos na pasta Drive
  - `selectNextCreative(used: string[])` — seleciona o próximo criativo não testado
  - `generateCreativeCopy(ctx, adData)` — gera nova copy via LLM
  - `buildCreativeProposal(ad, creative, copy)` — monta a mensagem de proposta
- Handler de aprovação `/aprovar_criativo_[hash]` no Telegram bot
- Integração com `atlas-notifier.ts` para envio da proposta
- Novo env var `ATLAS_CREATIVE_DRIVE_FOLDER_ID`

**OUT:**
- Não alterar o fluxo de análise principal (mantém `atlasAutoAnalyze()`)
- Não criar novos endpoints (fluxo via Telegram bot existente)
- Não implementar upload de criativo à Meta nesta story (apenas proposta)
- Implementação da Meta Creative API (Story 8.6 — futuro)

## Dependencies

- **Story 8.4** deve estar concluída (sistema de notificação Telegram existe)
- **Google Workspace** integração já configurada (Gmail, Calendar existentes)
- `ATLAS_CREATIVE_DRIVE_FOLDER_ID` configurado no `.env`

## Complexity Estimate

**T-Shirt:** Medium (M)
**Story Points:** 5
**Dev Time:** ~6 horas

## Business Value

Fecha o loop completo de otimização autônoma: detectar problema de criativo → buscar substituto → propor com nova copy → executar com aprovação. Elimina o trabalho manual de troca de criativos e garante que o operador tenha controle total via Telegram.

## Risks

- **Risco:** Google Drive API pode retornar formatos de arquivo não suportados pela Meta
- **Mitigação:** Filtrar por extensões suportadas (.jpg, .png, .mp4, .mov)
- **Risco:** LLM pode gerar copy genérica ou inadequada
- **Mitigação:** Incluir histórico de performance e briefing do produto no prompt de geração de copy
