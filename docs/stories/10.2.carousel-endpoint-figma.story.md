# Story 10.2: Maverick — Endpoint de Carrossel e Integração Figma

## Status
Ready for Dev

## Story

**As a** usuário do Maverick,
**I want** um endpoint que gere a estrutura de carrossel e opcionalmente exporte para o Figma,
**so that** eu receba tanto o JSON estruturado quanto um link para o design.

## Acceptance Criteria

1. **Endpoint:** `POST /api/maverick/carousel` aceita `{ script: ScriptData, exportToFigma?: boolean }` e retorna `{ carousel: CarouselStructure, figmaUrl?: string }`.
2. **Sem Figma configurado:** Se `FIGMA_API_TOKEN` não estiver no `.env` ou `exportToFigma` for `false`, retorna apenas o JSON da estrutura (sem erro).
3. **Com Figma configurado:** Se `exportToFigma: true` e `FIGMA_API_TOKEN` e `FIGMA_FILE_KEY` estiverem configurados, usa a Figma API para criar uma seção/página no arquivo com os frames dos slides e retorna a URL.
4. **Limitação técnica — Figma REST API:** A Figma REST API v1 **não suporta criação de conteúdo em arquivos de design** (é somente leitura). Por isso, a integração Figma desta story usa a abordagem de **exportar para um arquivo de texto Figma via Plugin API** OU gerar um link de template Figma pré-configurado. **Alternativa aprovada:** Criar um arquivo HTML exportável com o layout dos slides que pode ser importado no Figma via plugin "HTML to Figma".
5. **Fallback HTML:** Sempre que a exportação Figma não for possível, o endpoint inclui `htmlExport: string` — HTML completo representando os slides do carrossel, formatado para impressão/visualização.
6. **Feature flag:** `MAVERICK_CAROUSEL_ENABLED` no `.env` — se `false`, endpoint retorna 503.

## Scope

**IN:**
- Novo endpoint `POST /api/maverick/carousel` em `maverick.routes.ts`
- Gerador de HTML de carrossel (função `generateCarouselHtml(carousel: CarouselStructure): string`)
- Integração com Figma API (GET /v1/files para verificar acesso, depois abordagem de exportação)
- Feature flag `MAVERICK_CAROUSEL_ENABLED`

**OUT:**
- Não criar Figma Plugin (fora do escopo — requer conta de desenvolvedor Figma)
- Não criar frames diretamente via Figma REST API (tecnicamente impossível sem plugin)
- Não alterar o pipeline de SSE existente (isso é Story 10.3)

## Dependencies

## Tasks / Subtasks

- [ ] **Task 1** — Criar `generateCarouselHtml()` em `carousel-designer/html-export.ts` (AC: 5)
  ```ts
  // Gera HTML com cada slide como um div 1080x1080px
  // Inclui: título, body, visual_hint, número do slide
  // Estilizado com CSS inline para fácil cópia/visualização
  export function generateCarouselHtml(carousel: CarouselStructure): string {
    const slides = carousel.slides.map(slide => `
      <div class="slide slide-${slide.type}" style="...">
        <div class="slide-number">${slide.position}/${carousel.total_slides}</div>
        <h2>${slide.title}</h2>
        <p>${slide.body}</p>
        <small class="visual-hint">🎨 ${slide.visual_hint}</small>
      </div>
    `).join('\n');

    return `<!DOCTYPE html><html>...${slides}...</html>`;
  }
  ```

- [ ] **Task 2** — Criar endpoint `POST /api/maverick/carousel` (AC: 1, 2, 6)
  ```ts
  fastify.post('/carousel', async (req, reply) => {
    if (process.env.MAVERICK_CAROUSEL_ENABLED === 'false') {
      return reply.status(503).send({ error: 'Carousel feature not enabled' });
    }

    const { script, exportToFigma = false } = req.body as any;
    if (!script) return reply.status(400).send({ error: 'script é obrigatório' });

    const carousel = generateCarouselStructure(script);
    const htmlExport = generateCarouselHtml(carousel);

    let figmaUrl: string | undefined;

    if (exportToFigma && process.env.FIGMA_API_TOKEN && process.env.FIGMA_FILE_KEY) {
      figmaUrl = await tryFigmaExport(carousel);
    }

    return reply.send({ carousel, htmlExport, figmaUrl });
  });
  ```

- [ ] **Task 3** — Implementar `tryFigmaExport()` com Figma API (AC: 3, 4)
  ```ts
  async function tryFigmaExport(carousel: CarouselStructure): Promise<string | undefined> {
    try {
      const token = process.env.FIGMA_API_TOKEN!;
      const fileKey = process.env.FIGMA_FILE_KEY!;

      // Verificar acesso ao arquivo (GET é suportado pela API)
      const res = await fetch(`https://api.figma.com/v1/files/${fileKey}`, {
        headers: { 'X-Figma-Token': token },
      });

      if (!res.ok) return undefined;

      // Figma REST API não suporta criação de nós via REST.
      // Retornamos a URL do arquivo para o designer abrir e aplicar manualmente.
      // Em versões futuras: usar Figma Plugin API ou "Content Reel" plugin.
      return `https://www.figma.com/file/${fileKey}`;
    } catch {
      return undefined;
    }
  }
  ```
  **Nota:** Esta função retorna a URL do arquivo Figma existente como ponto de partida. A criação real de slides no Figma requer um plugin (fora do escopo desta story). O HTML export é o entregável principal.

## Dev Notes

### Por que não criar frames via Figma REST API?

A Figma REST API v1 permite apenas:
- GET: ler arquivos, componentes, estilos, exportar imagens
- POST: somente comments e webhooks

Para criar/modificar conteúdo em arquivos Figma programaticamente, as opções são:
1. **Figma Plugin API** — roda dentro do app Figma, requer plugin publicado
2. **Figma Variables API** (beta) — apenas variables, não frames
3. **"HTML to Figma" plugin** — plugin da comunidade que importa HTML

A abordagem desta story (HTML export) é a mais prática para o caso de uso atual.
