## Story

**As a** usuário,
**I want** acessar uma interface web protegida por senha e conversar com a ARIA com streaming de respostas,
**so that** eu tenha uma alternativa premium ao Telegram no desktop com identidade visual ARIA e autenticação JWT.

## Acceptance Criteria

1. Aplicação Next.js 15 com tela de login protegida — autenticação via JWT com senha master configurada em `.env` (`WEB_PASSWORD`)
2. Após login, JWT armazenado em **httpOnly cookie** (`aria-token`) com expiração de 7 dias
3. Interface de chat com: lista de mensagens rolável, campo de input, botão de envio, suporte a Enter para enviar
4. Mensagens enviadas chegam ao `apps/api` via `POST /api/chat/stream` com SSE — resposta exibida em streaming (caractere a caractere)
5. Indicador visual de "ARIA está processando..." durante chamada ao backend
6. Upload de arquivo (PDF/Word) visível no input (botão de clipe) — **sem processamento** (placeholder para Epic 2)
7. Interface **responsiva** para desktop e mobile (mobile-first com TailwindCSS)
8. **Dark mode como padrão visual** — fundo `#050508`, acentos `#6366F1` (índigo)
9. Rota `/` redireciona para `/login` se não autenticado; redireciona para `/` se já autenticado e tentar acessar `/login`
10. Mensagens renderizadas com **Markdown** (negrito, itálico, listas, código)

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **JWT token expires mid-session** | User logged out unexpectedly | Low | Show notification 5 min before expiry |
| **SSE connection drops** | User loses stream | Medium | Reconnect with session ID fallback |
| **Markdown rendering XSS** | Security vulnerability | Low | Use `react-markdown` with `remark-gfm` sanitization |
| **Mobile responsiveness broken** | UI unusable on mobile | Low | Test on 3 screen sizes before marking done |

## Out of Scope

- ❌ **Social login** — Password only
- ❌ **File upload** — Placeholder button only
- ❌ **Conversation persistence** — Session memory only
- ❌ **Multi-user support** — Single user per instance

### Implementação de Referência — Next.js Middleware

```typescript
// apps/web/src/middleware.ts
import { NextRequest, NextResponse } from 'next/server';
import { jwtVerify } from 'jose';

const JWT_SECRET = new TextEncoder().encode(process.env.JWT_SECRET!);

export async function middleware(req: NextRequest) {
  const token = req.cookies.get('aria-token')?.value;
  const isLoginPage = req.nextUrl.pathname === '/login';

  if (!token) {
    if (isLoginPage) return NextResponse.next();
    return NextResponse.redirect(new URL('/login', req.url));
  }

  try {
    await jwtVerify(token, JWT_SECRET);
    if (isLoginPage) return NextResponse.redirect(new URL('/', req.url));
    return NextResponse.next();
  } catch {
    const res = NextResponse.redirect(new URL('/login', req.url));
    res.cookies.delete('aria-token');
    return res;
  }
}

export const config = {
  matcher: ['/((?!api|_next/static|_next/image|favicon.ico).*)'],
};
```

### Implementação de Referência — chat.service.ts (SSE)

```typescript
// apps/web/src/services/chat.service.ts
export async function* streamMessage(
  content: string,
  sessionId: string
): AsyncGenerator<string> {
  const apiUrl = process.env.NEXT_PUBLIC_API_URL ?? 'http://localhost:3001';

  const response = await fetch(`${apiUrl}/api/chat/stream`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    credentials: 'include', // envia cookie aria-token
    body: JSON.stringify({ content, sessionId }),
  });

  if (!response.ok) throw new Error(`API error: ${response.status}`);

  const reader = response.body!.getReader();
  const decoder = new TextDecoder();

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    const chunk = decoder.decode(value, { stream: true });
    for (const line of chunk.split('\n')) {
      if (!line.startsWith('data: ')) continue;
      const data = JSON.parse(line.slice(6));
      if (data.type === 'chunk') yield data.content as string;
      if (data.type === 'done') return;
    }
  }
}
```

### Implementação de Referência — chatStore (Zustand)

```typescript
// apps/web/src/stores/chatStore.ts
import { create } from 'zustand';
import type { Message } from '@aria/shared';

interface ChatStore {
  messages: Message[];
  isStreaming: boolean;
  streamingContent: string;
  addMessage: (msg: Message) => void;
  setStreaming: (v: boolean) => void;
  appendStreamChunk: (chunk: string) => void;
  commitStreamedMessage: () => void;
}

export const useChatStore = create<ChatStore>((set) => ({
  messages: [],
  isStreaming: false,
  streamingContent: '',

  addMessage: (msg) => set((s) => ({ messages: [...s.messages, msg] })),
  setStreaming: (v) => set({ isStreaming: v }),
  appendStreamChunk: (chunk) => set((s) => ({ streamingContent: s.streamingContent + chunk })),
  commitStreamedMessage: () =>
    set((s) => ({
      messages: [
        ...s.messages,
        {
          id: crypto.randomUUID(),
          role: 'assistant' as const,
          content: s.streamingContent,
          timestamp: new Date(),
          sessionId: '',
          contentType: 'text' as const,
        },
      ],
      streamingContent: '',
      isStreaming: false,
    })),
}));
```
