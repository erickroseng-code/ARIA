## Story

**As a** desenvolvedor,
**I want** um monorepo configurado com TypeScript, Turborepo, linting e variáveis de ambiente,
**so that** o projeto ARIA tenha uma base consistente, escalável e compartilhável entre todos os apps desde o início.

## Acceptance Criteria

1. Monorepo criado com estrutura `apps/web`, `apps/bot`, `apps/api`, `packages/core`, `packages/integrations`, `packages/shared`
2. TypeScript configurado em todos os pacotes com `tsconfig.base.json` base compartilhado e `tsconfig.json` individuais em cada app/package
3. Turborepo configurado (`turbo.json`) com pipeline para `build`, `dev`, `lint`, `test`, `typecheck`
4. ESLint + Prettier configurados com regras consistentes compartilhadas entre todos os pacotes
5. Arquivo `.env.example` documentando todas as variáveis necessárias (listadas em Dev Notes)
6. Scripts npm funcionais na raiz: `dev`, `dev:web`, `dev:api`, `dev:bot`, `build`, `lint`, `test`, `typecheck`
7. `package.json` raiz com `"workspaces": ["apps/*", "packages/*"]` e dependências dev compartilhadas
8. Cada `packages/*` com seu `package.json` e nome de escopo `@aria/*` (ex: `@aria/core`, `@aria/shared`)

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Workspace dependency conflicts** | Build fails across packages | Medium | Use consistent versions in root; test `npm install` |
| **TypeScript config inheritance issues** | Type errors in child packages | Low | Validate all `tsconfig.json` extend base correctly |
| **Turborepo pipeline not optimized** | Slow builds in future stories | Low | Document pipeline in Dev Notes; tune later if needed |
| **ESLint conflicts with Prettier** | Formatting errors on lint | Low | Use `eslint-config-prettier` to disable conflicting rules |
