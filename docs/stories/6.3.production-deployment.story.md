# Story 6.3: Production Deployment & Infrastructure

## Status
Done

## Story

**As a** devops,
**I want** configurar CI/CD pipeline e deploy automático para produção,
**so that** releases sejam confiáveis, rápidas e repetíveis.

## Acceptance Criteria

1. Docker: Dockerfile for API server + Web UI (multi-stage builds)
2. CI/CD: GitHub Actions workflow (lint, test, build, deploy)
3. Environment management: .env.production, secrets via GitHub Secrets
4. Database migrations: automated during deployment
5. Deployment target: Railway or similar (easy one-click deploy)
6. Health checks: endpoint /health returns 200 when ready
7. Rollback strategy: automatic rollback on deployment failure
8. Monitoring: basic error tracking (Sentry) + logs (CloudFlare/Datadog)
9. SSL/TLS: automatic certificate via Let's Encrypt
10. Performance: deployment <5 minutes, zero-downtime updates
