## Story

**As a** usuário,
**I want** criar, consultar e cancelar eventos no Google Calendar via comando,
**so that** eu possa gerenciar minha agenda integralmente dentro da ARIA.

## Acceptance Criteria

1. Google Calendar API integrada via OAuth2 (user consent flow)
2. Criação de evento: "agenda reunião com Empresa X na quinta às 14h" → parsed, validated, created
3. Consulta: "o que tenho hoje?", "quais reuniões tenho essa semana?" → calendar events listed
4. Evento criado com: título, data, hora, duração padrão 1h (ajustável via parâmetro)
5. Confirmação retorna link direto: "📅 Reunião com *Empresa X* agendada para quinta, 14h — [Google Calendar link]"
6. Cancelamento por comando: "cancela reunião com X" → event removed from calendar
7. Suporte automático a timezone PT-BR (from user settings + system timezone)
8. OAuth2 refresh token rotation — no manual re-auth prompts
9. Calendar queries cached 5min — <2s response time
10. Error handling: rate limits → graceful degradation; OAuth expired → clear message with re-auth link

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **OAuth token expiration** | User loses calendar access | High | Implement refresh token rotation; check token validity on each call; clear error message |
| **User not authenticated with Google** | Can't create calendar event | Medium | Show setup link to Google account OAuth; graceful error message |
| **Timezone conflicts** | Wrong time displayed; events scheduled wrong time | Medium | Use user's timezone from account settings + validate at creation; test DST transitions |
| **Calendar API rate limits (100 req/min)** | Requests blocked; operations fail | Low | Implement 5-min caching; batch requests; queue system from Epic 3 reusable |
| **Google Calendar unavailable** | Operations timeout | Low | 10s timeout + queue for retry (Story 3.2 pattern) |

## Out of Scope

- ❌ Calendar sharing (single user only for MVP)
- ❌ Invite attendees (create event for user only)
- ❌ Recurring events (one-off events only)
- ❌ Video meeting links (manual Google Meet setup)
- ❌ Multiple calendars (primary calendar only; extensible Phase 2)
- ❌ Complex event editing (create/cancel only; edit via Google Calendar)

## Dev Notes

### Dependencies
- Depends on Story 3.4: IntentParser provides structured command data
- Parallel with Story 4.2: Both use calendar event data

### Architecture Reference
- API endpoint: `POST /api/google-calendar/events` (create), `GET /api/google-calendar/events` (query)
- OAuth2 flow: Authorization Code flow with refresh token rotation
- Caching: 5-min TTL for query results (configurable)
- Error handling: Timeout 10s per API call; queue system for retries
- Timezone: User's system timezone + explicit PT-BR support
