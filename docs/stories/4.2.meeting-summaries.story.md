## Story

**As a** usuário,
**I want** gerar resumos automáticos de minhas reuniões via IA,
**so that** eu possa consultar rapidamente o que foi discutido sem ler transcrições completas.

## Acceptance Criteria

1. Detect quando reunião foi concluída no Google Calendar (time-based trigger + user indicator)
2. Accept meeting notes/transcript (user can paste text, upload file, or transcription from Story 3.1)
3. Claude Haiku generates summary (5-10 bullet points): participants, decisions, action items, next steps
4. Summary stored in Notion as child block under "📋 Meeting Summary" section
5. Summary generated within 5 minutes of meeting end
6. User notified when summary ready (Telegram: "✅ Resumo pronto"; Web: banner notification)
7. Summary format: structured JSON → converted to markdown blocks in Notion
8. Error handling: if Claude fails → queue for retry; user informed of delay
9. Support multiple input formats: text paste, file upload (PDF/DOCX), audio transcription
10. Extensible: add support for Google Meet transcript auto-fetch in Phase 2

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Claude API latency (1-2 min)** | User waits; timeout risk | Medium | Async queue; user notified when ready; timeout 3 min |
| **Meeting detection fails** | Summary not generated | Low | Manual trigger: user command "resumir reunião"; fallback to time-based |
| **Input file parsing fails** | Summary can't be generated | Low | Graceful error; ask user to paste text instead |
| **Notion storage fails** | Summary not persisted | Low | Queue system + retry; store in Redis as backup |
| **Rate limits (Claude/Notion)** | Operations blocked | Low | Queue system reusable; batch summarization |

## Out of Scope
