# Story 8.3: Atlas — Audit Log de Ações Autônomas

## Status
Ready for Dev

## Story

**As a** operador da Aria,
**I want** que cada ação executada pelo Atlas seja registrada numa tabela SQLite com timestamp, contexto e resultado,
**so that** eu possa auditar o que o Atlas fez autonomamente e detectar anomalias.

## Acceptance Criteria

1. **Tabela SQLite:** `atlas_audit_log` criada no startup da API se não existir. Schema:
   ```sql
   CREATE TABLE IF NOT EXISTS atlas_audit_log (
     id INTEGER PRIMARY KEY AUTOINCREMENT,
     timestamp TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
     action TEXT NOT NULL,
     entity_type TEXT NOT NULL,  -- 'campaign' | 'adset' | 'ad'
     entity_id TEXT NOT NULL,
     workspace_id TEXT NOT NULL,
     dry_run INTEGER NOT NULL DEFAULT 1,
     result TEXT NOT NULL,
     reason TEXT,
     triggered_by TEXT NOT NULL DEFAULT 'scheduler'  -- 'scheduler' | 'chat'
   )
   ```
2. **Registro automático:** `executeAction()` (função refatorada na Story 8.2) recebe um logger opcional. Quando fornecido, registra a ação no `atlas_audit_log` após execução.
3. **Endpoint de histórico:** `GET /api/traffic/atlas/audit?limit=50&workspaceId=default` retorna array de logs mais recentes.
4. **Registro de erros:** Se a ação falhar, o campo `result` inclui a mensagem de erro (ex: `"ERROR: API timeout"`).
5. **Sem PII nem secrets:** `result` e `reason` nunca devem conter access tokens, credenciais ou dados pessoais.

## Scope

**IN:**
- Criar tabela `atlas_audit_log` em `db` (SQLite nativo, mesmo padrão do resto da API)
- Adicionar função `logAtlasAction(entry)` em novo arquivo `atlas-audit.ts`
- Passar logger para `executeAction()` no scheduler e opcionalmente no chat
- Endpoint `GET /api/traffic/atlas/audit`

**OUT:**
- Não alterar schema existente (`integrations`, outras tabelas)
- Não implementar retenção automática de logs (pode ser feito depois)
- Não integrar com sistemas externos de log

## Dependencies

- **Story 8.2** deve estar concluída (função `executeAction()` refatorada já existe)

## Complexity Estimate

**T-Shirt:** Small (S)
**Story Points:** 3
**Dev Time:** ~3 horas

## Business Value

Auditabilidade é crítica para operações financeiras autônomas. Sem log, impossível saber o que o Atlas fez, quando, e por quê.

## Risks

- **Risco:** Erro no log não deve impedir a execução da ação principal
- **Mitigação:** `logAtlasAction()` tem try/catch — erros de log são ignorados silenciosamente (apenas `console.error`)

- [ ] **Task 2** — Criar `atlas-audit.ts` com `logAtlasAction()` (AC: 2, 4)
  ```ts
  // aria/apps/api/src/modules/traffic/agents/atlas-audit.ts
  import { db } from '../../../config/db';

  export interface AuditEntry {
    action: string;
    entityType: 'campaign' | 'adset' | 'ad' | 'unknown';
    entityId: string;
    workspaceId: string;
    dryRun: boolean;
    result: string;
    reason?: string;
    triggeredBy?: 'scheduler' | 'chat';
  }

  export function logAtlasAction(entry: AuditEntry): void {
    try {
      db.prepare(`
        INSERT INTO atlas_audit_log
          (action, entity_type, entity_id, workspace_id, dry_run, result, reason, triggered_by)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?)
      `).run(
        entry.action,
        entry.entityType,
        entry.entityId,
        entry.workspaceId,
        entry.dryRun ? 1 : 0,
        entry.result,
        entry.reason ?? null,
        entry.triggeredBy ?? 'scheduler'
      );
    } catch (err) {
      console.error('[Atlas Audit] Failed to log action:', err);
    }
  }
  ```

- [ ] **Task 3** — Integrar `logAtlasAction()` com `executeAction()` em `atlas-orchestrator.ts` (AC: 2, 4)
  - Modificar a assinatura de `executeAction()` para aceitar `logContext?: { workspaceId: string; triggeredBy: 'scheduler' | 'chat' }`
  - Após executar cada ação (ou capturar erro), chamar `logAtlasAction()`
  - `atlasSchedulerRun()` passa `{ workspaceId: ctx.workspaceId, triggeredBy: 'scheduler' }`
  - `atlasChat()` pode passar `{ triggeredBy: 'chat' }` (opcional — sem workspaceId pode usar `ctx.workspace`)

- [ ] **Task 4** — Endpoint `GET /api/traffic/atlas/audit` em `traffic.routes.ts` (AC: 3)
  ```ts
  fastify.get('/atlas/audit', async (req, reply) => {
    const { limit = 50, workspaceId } = req.query as any;
    const query = workspaceId
      ? db.prepare('SELECT * FROM atlas_audit_log WHERE workspace_id = ? ORDER BY id DESC LIMIT ?')
          .all(workspaceId, Math.min(Number(limit), 200))
      : db.prepare('SELECT * FROM atlas_audit_log ORDER BY id DESC LIMIT ?')
          .all(Math.min(Number(limit), 200));
    return reply.send({ logs: query });
  });
  ```

  if (action.includes('adset')) return 'adset';
