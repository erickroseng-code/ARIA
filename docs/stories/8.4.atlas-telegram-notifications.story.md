# Story 8.4: Atlas — Notificações Telegram após Otimizações Autônomas

## Status
Ready for Dev

## Story

**As a** operador da Aria,
**I want** receber uma mensagem no Telegram após cada ciclo de otimização autônoma do Atlas,
**so that** eu saiba o que foi feito sem precisar verificar logs manualmente.

## Acceptance Criteria

1. **Notificação pós-scheduler:** Ao final de `atlasSchedulerRun()`, se houver ações executadas (não dry-run), enviar mensagem Telegram com resumo.
2. **Formato da mensagem:**
   ```
   🤖 *Atlas — Ciclo de Otimização*
   📅 10/03/2026 14:00

   ✅ Campanha "Nome" pausada (CTR 0.2%)
   ✅ Budget do Conjunto "Nome" reduzido -15% (CPC R$3.20 → meta R$2.50)
   ❌ Erro ao pausar anúncio 123: API timeout

   Total: 2 ações | 1 erro
   Modo: PRODUÇÃO (dry_run=false)
   ```
3. **Dry-run silencioso:** Em modo dry-run, não enviar mensagem Telegram (evitar spam).
4. **Env vars:** `TELEGRAM_BOT_TOKEN` e `TELEGRAM_CHAT_ID` já existem no `.env`. Usar esses valores.
5. **Falha tolerante:** Se envio Telegram falhar, logar erro mas não travar o endpoint (ação já executada e logada).
6. **Notificação de erro crítico:** Se o scheduler encontrar erro antes de executar qualquer ação (ex: falha ao buscar campanhas), também enviar mensagem de alerta:
   ```
   ⚠️ *Atlas — Erro no Ciclo*
   Falha ao carregar dados: [mensagem do erro]
   ```

## Scope

**IN:**
- Criar `atlas-notifier.ts` com `sendAtlasTelegramNotification()`
- Integrar notificação no endpoint `POST /api/traffic/atlas/auto-optimize`
- Usar `TELEGRAM_BOT_TOKEN` e `TELEGRAM_CHAT_ID` existentes

**OUT:**
- Não criar novo bot Telegram
- Não enviar notificações do chat interativo (`atlasChat()`) — apenas do scheduler
- Não criar sistema de preferências/configuração de notificações

## Dependencies

- **Story 8.2** deve estar concluída (endpoint do scheduler existe)
- **Story 8.3** deve estar concluída (audit log existe para referência no summary)

## Complexity Estimate

**T-Shirt:** Small (S)
**Story Points:** 2
**Dev Time:** ~2 horas

## Business Value

Sem notificação, o operador precisa verificar ativamente se o Atlas agiu. Com Telegram, o loop de controle fica fechado: GitHub Actions dispara → Atlas age → operador vê no Telegram.

## Risks

- **Risco:** Telegram API fora do ar → scheduler falha
- **Mitigação:** try/catch isolado, ação principal já executada antes do notify

## Tasks / Subtasks

- [ ] **Task 1** — Criar `atlas-notifier.ts` (AC: 1, 2, 3, 4, 5)
  ```ts
  // aria/apps/api/src/modules/traffic/agents/atlas-notifier.ts

  interface ActionResult {
    action: string;
    entityId: string;
    result: string;
    dryRun: boolean;
  }

  export async function sendAtlasNotification(
    actionsExecuted: ActionResult[],
    dryRun: boolean
  ): Promise<void> {
    if (dryRun) return; // Silencioso em dry-run

    const botToken = process.env.TELEGRAM_BOT_TOKEN;
    const chatId = process.env.TELEGRAM_CHAT_ID;
    if (!botToken || !chatId) {
      console.warn('[Atlas Notify] TELEGRAM_BOT_TOKEN ou TELEGRAM_CHAT_ID não configurados');
      return;
    }

    const now = new Date().toLocaleString('pt-BR', { timeZone: 'America/Sao_Paulo' });
    const lines = [`🤖 *Atlas — Ciclo de Otimização*`, `📅 ${now}`, ''];

    for (const a of actionsExecuted) {
      const icon = a.result.startsWith('ERROR') ? '❌' : '✅';
      lines.push(`${icon} ${a.action} (${a.entityId}): ${a.result}`);
    }

    const errors = actionsExecuted.filter(a => a.result.startsWith('ERROR')).length;
    lines.push('');
    lines.push(`Total: ${actionsExecuted.length} ações | ${errors} erro(s)`);
    lines.push(`Modo: ${dryRun ? 'DRY-RUN' : 'PRODUÇÃO'}`);

    const text = lines.join('\n');

    try {
      await fetch(`https://api.telegram.org/bot${botToken}/sendMessage`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ chat_id: chatId, text, parse_mode: 'Markdown' }),
      });
    } catch (err) {
      console.error('[Atlas Notify] Falha ao enviar Telegram:', err);
    }
  }

  export async function sendAtlasErrorAlert(errorMessage: string): Promise<void> {
    const botToken = process.env.TELEGRAM_BOT_TOKEN;
    const chatId = process.env.TELEGRAM_CHAT_ID;
    if (!botToken || !chatId) return;

    try {
      await fetch(`https://api.telegram.org/bot${botToken}/sendMessage`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          chat_id: chatId,
          text: `⚠️ *Atlas — Erro no Ciclo*\nFalha ao carregar dados: ${errorMessage}`,
          parse_mode: 'Markdown',
        }),
      });
    } catch { /* ignorar */ }
  }
  ```
