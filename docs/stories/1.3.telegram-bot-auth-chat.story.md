## Story

**As a** usuário,
**I want** enviar mensagens ao bot do Telegram e receber respostas da ARIA via IA,
**so that** eu possa usar o Telegram como canal de comando com garantia de que apenas meu ID autorizado consegue acessar o bot.

## Acceptance Criteria

1. Telegram Bot conectado via **polling** em desenvolvimento e **webhook** em produção — controlado por `NODE_ENV`
2. Whitelist de Telegram IDs configurável via `.env` (`ALLOWED_TELEGRAM_IDS`) — NFR9
3. Mensagens de IDs **não autorizados ignoradas silenciosamente** (sem resposta, sem log de conteúdo)
4. Mensagem de texto qualquer → ChatService → resposta enviada de volta no chat Telegram (formato MarkdownV2)
5. Comando `/start` retorna mensagem de boas-vindas da ARIA com introdução e lista de capacidades
6. Comando `/help` lista os comandos disponíveis com descrição curta
7. Cada chat (`chat_id`) tem sua própria sessão isolada com `sessionId` único
8. Erros da Claude API tratados graciosamente — usuário recebe mensagem de erro amigável em português

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Invalid Telegram IDs in whitelist** | Authorized users blocked | Low | Validate IDs at startup; log misconfigurations |
| **Polling lag causes missed messages** | User experiences delay | Low | Use appropriate polling interval |
| **Webhook routing issues** | Messages not received in prod | Low | Test webhook with Telegram test bot first |
| **Memory leak in session cache** | Bot memory grows | Medium | Implement session cleanup (Story 1.5) |

## Out of Scope

- ❌ **Media handling** — Text messages only (documents in Story 2.2)
- ❌ **Group chat support** — Private chat only
- ❌ **Inline keyboard** — Text commands only

### Implementação de Referência — bot.ts

```typescript
// apps/bot/src/bot.ts
import { Bot } from 'grammy';
import { env } from './config/env';
import { whitelistMiddleware } from './middleware/whitelist.middleware';
import { sessionMiddleware } from './middleware/session.middleware';
import { startHandler, helpHandler } from './handlers/command.handler';
import { messageHandler } from './handlers/message.handler';

export const bot = new Bot(env.TELEGRAM_BOT_TOKEN);

// Middlewares (ordem importa!)
bot.use(whitelistMiddleware(env.ALLOWED_TELEGRAM_IDS));
bot.use(sessionMiddleware);

// Comandos
bot.command('start', startHandler);
bot.command('help', helpHandler);

// Mensagens
bot.on('message:text', messageHandler);

// Inicialização
if (env.NODE_ENV !== 'production') {
  bot.start(); // Polling em dev
  console.log('Bot running in polling mode...');
}
```

### Implementação de Referência — whitelist middleware

```typescript
// apps/bot/src/middleware/whitelist.middleware.ts
import { MiddlewareFn, Context } from 'grammy';

export function whitelistMiddleware(allowedIds: number[]): MiddlewareFn<Context> {
  return async (ctx, next) => {
    const userId = ctx.from?.id;

    // ID ausente ou não autorizado: ignorar silenciosamente
    if (!userId || !allowedIds.includes(userId)) {
      return; // Sem resposta, sem log de conteúdo
    }

    return next();
  };
}
```

### Implementação de Referência — message handler

```typescript
// apps/bot/src/handlers/message.handler.ts
import { Context } from 'grammy';
import { chatService } from '../services/chat';
import { ERROR_MESSAGE } from '../templates/responses';
import { logger } from '@aria/shared'; // ou pino local

export async function messageHandler(ctx: Context) {
  const text = ctx.message?.text;
  const sessionId = ctx.session.sessionId;

  if (!text) return;

  // Log sem conteúdo sensível (NFR13)
  logger.info({ sessionId, userId: ctx.from?.id }, 'Telegram message received');

  try {
    const response = await chatService.completeResponse(text, sessionId);
    await ctx.reply(response, { parse_mode: 'MarkdownV2' });
  } catch (error) {
    logger.error({ err: error, sessionId }, 'ChatService error');
    await ctx.reply(ERROR_MESSAGE);
  }
}
```

### CRÍTICO: Escape MarkdownV2

O Telegram MarkdownV2 exige escape de caracteres especiais. Qualquer `.`, `!`, `-`, `(`, `)`, `[`, `]`, `{`, `}`, `+`, `=`, `|`, `#`, `>`, `~`, `\` na mensagem deve ser prefixado com `\`.

**Em respostas geradas pela IA (Claude):** considerar usar `parse_mode: undefined` (texto plano) ou fazer sanitização antes de enviar — Claude pode gerar conteúdo com esses caracteres.

Alternativa recomendada para respostas dinâmicas da IA:
```typescript
// Sanitizar resposta da IA antes de enviar como MarkdownV2
function escapeMarkdownV2(text: string): string {
  return text.replace(/[_*[\]()~`>#+=|{}.!\\-]/g, '\\$&');
}
await ctx.reply(escapeMarkdownV2(response), { parse_mode: 'MarkdownV2' });
```

### SessionId por Canal

```
Telegram: sessionId = "tg_{chat_id}"    → ex: "tg_123456789"
Web UI:   sessionId = "web_{userId}"   → ex: "web_abc123"
```

Isso garante que o `ContextStore` (Story 1.2) mantém histórico separado por canal e usuário.
