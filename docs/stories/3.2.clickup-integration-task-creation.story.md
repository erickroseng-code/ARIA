## Story

**As a** usuário,
**I want** criar tarefas no ClickUp via texto ou áudio natural,
**so that** eu possa capturar ações e compromissos imediatamente — sem abrir o ClickUp manualmente — com a ARIA organizando título, prazo e prioridade automaticamente.

## Acceptance Criteria

1. Integração com ClickUp REST API v2 configurada via `CLICKUP_API_KEY` e `CLICKUP_LIST_ID` no `.env`
2. Criação de tarefa com: título (obrigatório), descrição, data de vencimento e prioridade (opcionais)
3. Usuário pode especificar lista/projeto diferente da padrão: `"cria no projeto Marketing"` → busca lista pelo nome
4. Lista padrão configurável via `CLICKUP_LIST_ID` no `.env` — usada quando nenhum projeto especificado
5. Confirmação enviada: `"✅ Tarefa criada no ClickUp: *Ligar para Empresa X* — vence sexta-feira\n🔗 [Ver tarefa](https://app.clickup.com/t/...)"`
6. Link direto para a tarefa incluído na confirmação (campo `url` da resposta ClickUp)
7. Cliente ativo no contexto (`activeClientId`) associado à tarefa quando disponível — descrição inclui `"Cliente: {Nome}"`
8. Erros da API ClickUp tratados com mensagem amigável e código `CLICKUP_001`

### ClickUp API v2 — Referência

```typescript
// Criar tarefa
// POST https://api.clickup.com/api/v2/list/{list_id}/task
// Headers: { Authorization: process.env.CLICKUP_API_KEY, 'Content-Type': 'application/json' }

const body = {
  name: "Ligar para Empresa X",
  description: "Cliente: Empresa X\n\nLigar para confirmar reunião de onboarding.",
  due_date: 1740700800000,  // Unix ms — ATENÇÃO: em ms, não em segundos
  due_date_time: false,     // false = data sem hora específica
  priority: 2,              // 1=urgent, 2=high, 3=normal, 4=low
  status: "to do"
};

// Resposta inclui:
// { id: "abc123", name: "...", url: "https://app.clickup.com/t/abc123", ... }
```

**CRÍTICO — Header de autenticação ClickUp:**
```
Authorization: pk_12345...     ← SEM prefixo "Bearer"
```
(Diferente da maioria das APIs que usam `Authorization: Bearer token`)

### `ClickUpClient` — Implementação de Referência

```typescript
// packages/integrations/src/clickup/ClickUpClient.ts
export class ClickUpClient {
  private readonly BASE = 'https://api.clickup.com/api/v2';

  constructor(
    private readonly apiKey: string,
    private readonly defaultListId: string,
  ) {}

  private async request<T>(path: string, options?: RequestInit): Promise<T> {
    const res = await fetch(`${this.BASE}${path}`, {
      ...options,
      headers: {
        'Authorization': this.apiKey,  // Sem "Bearer"
        'Content-Type': 'application/json',
        ...options?.headers,
      },
    });
    if (!res.ok) {
      const body = await res.json().catch(() => ({}));
      throw new AppError('CLICKUP_001', body.err ?? `ClickUp API error ${res.status}`, res.status);
    }
    return res.json();
  }

  async createTask(listId: string, params: CreateTaskParams): Promise<ClickUpTask> {
    return this.request<ClickUpTask>(`/list/${listId}/task`, {
      method: 'POST',
      body: JSON.stringify({
        name: params.name,
        description: params.description,
        due_date: params.due_date,
        due_date_time: false,
        priority: params.priority ?? 3,  // normal como default
        status: params.status ?? 'to do',
      }),
    });
  }

  async findListByName(teamId: string, name: string): Promise<ClickUpList | null> {
    const { lists } = await this.request<{ lists: ClickUpList[] }>(`/team/${teamId}/list`);
    return lists.find(l => l.name.toLowerCase().includes(name.toLowerCase())) ?? null;
  }
}
```

### Formatação da Data em Português

```typescript
// packages/core/src/services/TaskService.ts
function formatDueDate(dueDateMs: string | null): string {
  if (!dueDateMs) return '';
  const date = new Date(Number(dueDateMs));
  const formatted = new Intl.DateTimeFormat('pt-BR', {
    weekday: 'long', day: 'numeric', month: 'long'
  }).format(date);
  return ` — vence ${formatted}`;  // ex: " — vence sexta-feira, 28 de março"
}
```
