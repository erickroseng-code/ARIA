## Story

**As a** usuário,
**I want** criar tarefas no ClickUp por comando de texto ou voz,
**so that** eu possa gerenciar tarefas diretamente pelo Telegram ou Web UI sem abrir o ClickUp.

## Acceptance Criteria

1. ClickUp API integrada via token configurável em `.env`
2. Tarefa criada com: título, descrição, data de vencimento, prioridade
3. Usuário pode especificar lista/projeto ("criar no projeto X")
4. Quando lista não especificada: usa lista padrão de `.env`
5. Tarefa criada em menos de 3 segundos
6. Confirmação retorna link direto para a tarefa no ClickUp
7. Formato de mensagem: "✅ Tarefa criada no ClickUp: *[título]* — vence [data] — [link]"
8. Rate limiting: máximo 30 requisições/minuto ao ClickUp (fila de retry)
9. Falha de criação: armazena em fila Redis, tenta novamente a cada 5 minutos (máx 3x)
10. Quando ClickUp falha: pergunta "Criar no Notion em vez disso?" (fallback)

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Rate limit hit (30 req/min)** | Task fails, goes to queue | Medium | Redis queue + exponential retry (5, 10, 15 min) |
| **Invalid list/project name** | Task creation fails | Low | Validate via ClickUp API before creation |
| **ClickUp API unavailable** | Task cannot be created | Low | Queue system + fallback to Notion option |
| **User creates duplicate task** | Same task created twice | Low | Deduplication via title+date hash (7-day window) |
