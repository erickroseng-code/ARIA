## Story

**As a** usuário,
**I want** criar tarefas e anotações no Notion via texto ou áudio,
**so that** eu possa registrar ações diretamente no Notion — especialmente vinculadas ao perfil de um cliente específico — mantendo toda a gestão no meu workspace Notion sem precisar alternar de ferramenta.

## Acceptance Criteria

1. Criação de item em database de tarefas do Notion configurável via `NOTION_TASKS_DATABASE_ID` no `.env`
2. Tarefa criada com: título (obrigatório), status `"A fazer"`, data de vencimento e prioridade (opcionais)
3. Suporte a vinculação de cliente: `"cria tarefa para Empresa X no Notion"` → tarefa com relação ao perfil do cliente
4. Quando vinculada a cliente: bloco `to_do` adicionado também dentro do perfil do cliente no Notion (seção "📋 Tarefas")
5. Confirmação: `"✅ Tarefa criada no Notion: *Preparar proposta* — vence segunda-feira\n🔗 [Ver tarefa](https://notion.so/...)"`
6. Se `NOTION_TASKS_DATABASE_ID` não configurado: fallback para criar bloco `to_do` diretamente no perfil do cliente (requer `activeClientId`)
7. Comando `"cria em ambos"` ou `destination: 'both'` → criar no ClickUp (Story 3.2) **e** no Notion simultaneamente
8. Retornar aviso se nem `NOTION_TASKS_DATABASE_ID` nem `activeClientId` disponíveis: `"⚠️ Configure uma database de tarefas no Notion ou selecione um cliente primeiro."`

## Complexity Estimate

**T-Shirt Sizing:** Medium (M)
**Story Points (if using):** 8-13 points
**Estimated Dev Time:** 3-4 days
**Estimated QA Time:** 1-2 days

**Complexity Drivers:**
- Multiple execution paths (database, fallback, both)
- Notion API integration (new, previously untested)
- Parallel execution (ClickUp + Notion simultaneously)
- Date/timezone handling complexity
- Fallback error handling

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Notion Database Schema Mismatch** | Database item creation fails silently | Medium | Validate schema properties at startup; log detailed errors; unit tests for all 5 property types |
| **Timezone Handling for Due Dates** | Incorrect date displayed to user (off by 1 day) | Medium | Convert `dueDateMs` → ISO 8601 at UTC; document timezone assumptions in code; test with multiple locales |
| **Fallback Complexity** | `to_do` blocks pollute client profiles; unclear hierarchy | Low | Add "📋 Tarefas" heading to organize; document fallback in user guide; limit to backup only |
| **ClickUp + Notion Partial Failure** | Confusing user message if one service fails | Low | Use `Promise.allSettled` to capture both; clear messaging for each success/failure |
| **Client Profile Relation Validation** | Invalid `clientPageId` causes errors | Medium | Validate page exists before creating relation; graceful error if page not found |
| **Missing NOTION_TASKS_DATABASE_ID Config** | User hits AC #8 warning without clear action | Low | Add `NOTION_TASKS_DATABASE_ID` to `.env.example` with explanation; suggest setup doc link in error |

## Out of Scope

The following features are **NOT** included in this story and will be addressed in future stories:

- ❌ **Task editing/updating** — Only creation (scope for Story 3.X)
- ❌ **Task deletion** — Planned for cleanup/maintenance story
- ❌ **Bulk task import** — Will be separate feature (Story X.X)
- ❌ **Notion database schema creation** — User must create manually (documented in `notion-tasks-database.md`)
- ❌ **Automatic Notion ↔ ClickUp sync** — One-way (ClickUp → Notion) only
- ❌ **Custom field mapping** — Fixed property names only (Nome, Status, Data, Prioridade, Cliente)
- ❌ **Notion webhook integration** — No bidirectional updates from Notion

## Dev Notes

### Notion API — Criar Item em Database

```typescript
// POST https://api.notion.com/v1/pages
// Headers: Authorization: Bearer {NOTION_API_KEY}, Notion-Version: 2022-06-28

const response = await notion.pages.create({
  parent: { database_id: process.env.NOTION_TASKS_DATABASE_ID },
  properties: {
    // Título (obrigatório)
    'Nome': {
      title: [{ text: { content: 'Preparar proposta para Empresa X' } }]
    },
    // Status
    'Status': {
      select: { name: 'A fazer' }
    },
    // Data de vencimento
    'Data': {
      date: { start: '2026-03-01' }
    },
    // Prioridade
    'Prioridade': {
      select: { name: 'Alta' }
    },
    // Relação com cliente (se vinculado)
    'Cliente': {
      relation: [{ id: 'notion_page_id_do_cliente' }]
    },
  }
});
// response.url → URL da página criada no Notion
```

**CRÍTICO:** Os nomes das propriedades (`'Nome'`, `'Status'`, etc.) devem corresponder **exatamente** aos nomes no database Notion. Case-sensitive. Se o usuário nomeou "Nome da Tarefa", usar esse nome.

### `NotionProps` Helper — Implementação

```typescript
// packages/integrations/src/notion/types.ts
export const NotionProps = {
  title: (text: string) => ({
    title: [{ type: 'text', text: { content: text } }]
  }),
  select: (name: string) => ({ select: { name } }),
  date: (start: string, end?: string) => ({
    date: { start, ...(end && { end }) }

    relation: [{ id: pageId }]
  }),
  richText: (text: string) => ({

};
```

### `destination: 'both'` — Execução Paralela

```typescript
// packages/core/src/services/TaskService.ts
const [clickupResult, notionResult] = await Promise.allSettled([
  this.createClickUpTask(entities, context),
  this.createNotionTask(entities, context),
]);

const clickupMsg = clickupResult.status === 'fulfilled'
  ? `• ClickUp: [Ver tarefa](${clickupResult.value.task.url})`
  : `• ClickUp: ⚠️ Falha (${(clickupResult.reason as Error).message})`;

const notionMsg = notionResult.status === 'fulfilled'
  ? `• Notion: [Ver tarefa](${notionResult.value.page.url})`
  : `• Notion: ⚠️ Falha (${(notionResult.reason as Error).message})`;

return {
  confirmationMessage: `✅ Tarefa criada em ambos:\n${clickupMsg}\n${notionMsg}`
};
```

### Schema Obrigatório do Database Notion (para setup)

O database `NOTION_TASKS_DATABASE_ID` deve ter as seguintes propriedades configuradas:

| Nome da Propriedade | Tipo Notion | Valores |
|--------------------|-------------|---------|
| `Nome` | Title | — |
| `Status` | Select | `A fazer`, `Em progresso`, `Concluída` |
| `Data` | Date | — |
| `Prioridade` | Select | `Urgente`, `Alta`, `Normal`, `Baixa` |
| `Cliente` | Relation | Apontar para database de clientes |

**Obter o Database ID:**
1. Abrir database no Notion
2. URL: `https://www.notion.so/{workspace}/{DATABASE_ID}?v=...`
3. O `DATABASE_ID` é o UUID de 32 chars antes do `?v=`
