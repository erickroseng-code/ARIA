## Story

**As a** usuário,
**I want** que informações extraídas dos documentos preencham automaticamente o cadastro do cliente no Notion,
**so that** eu não precise inserir manualmente os dados que a ARIA já identificou, mantendo o perfil do cliente sempre atualizado e com histórico completo das interações.

## Acceptance Criteria

1. Extração de metadados dos documentos analisados: responsável comercial, responsável marketing, metas principais, desafios, segmento de mercado
2. Propriedades do perfil do cliente atualizadas via Notion API após geração do Plano de Ataque (FR14)
3. Registro automático de histórico no perfil do cliente no Notion: data, tipo de operação, documentos processados e link da página gerada (FR15)
4. Propriedades já preenchidas no perfil não são sobrescritas automaticamente — usuário é informado e pode confirmar sobrescrita com "atualizar tudo"
5. Confirmação enviada ao usuário: "📋 Perfil de *Empresa X* atualizado com 4 novos campos"
6. Se houver campos com conflito: "⚠️ 2 campos já existiam e foram mantidos. Digite `atualizar tudo` para sobrescrever."

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Metadata extraction hallucination** | Wrong data in profile | Medium | Validate extracted fields; test with real analyses |
| **Property type mismatch** | Update fails silently | Low | Check property types match before update |
| **Conflict detection incorrect** | Data overwrites unexpectedly | Low | Test 10+ conflict scenarios |
| **Update partially fails** | Some fields not saved | Low | Use transaction-like flow or all-or-nothing |

## Out of Scope

- ❌ **Manual property mapping** — Auto-detect only
- ❌ **Custom properties** — Fixed set only
- ❌ **Batch updates** — Single client per flow
- ❌ **Rollback on error** — One-way update

### Notion API — Leitura de Propriedades

```typescript
// GET /v1/pages/{page_id}
const page = await notion.pages.retrieve({ page_id: pageId });
// page.properties contém todas as propriedades com seus valores atuais

// Verificar se propriedade rich_text está vazia:
const isEmpty = (prop: any): boolean => {
  if (prop.type === 'rich_text') return prop.rich_text.length === 0;
  if (prop.type === 'select') return prop.select === null;
  if (prop.type === 'multi_select') return prop.multi_select.length === 0;
  return true;
};
```

### `ClientProfileService` — Fluxo de Detecção de Conflitos

```typescript
interface FillResult {
  updated: string[];       // campos efetivamente atualizados no Notion
  conflicted: ConflictDetail[];  // campos com valor existente diferente do novo
  pageId: string;
}

interface ConflictDetail {
  field: string;           // ex: "responsavel_comercial"
  notionPropName: string;  // ex: "Responsável Comercial"
  existing: string;        // valor atual no Notion
  incoming: string;        // valor extraído dos documentos
}
```

### `CLIENT_PROPERTY_MAP` — Mapeamento Padrão

```typescript
// packages/integrations/src/notion/constants.ts
export const CLIENT_PROPERTY_MAP: Record<keyof ClientMetadata, {
  notionName: string;
  type: 'rich_text' | 'select' | 'multi_select';
}> = {
  responsavel_comercial: { notionName: 'Responsável Comercial', type: 'rich_text' },
  responsavel_marketing: { notionName: 'Responsável Marketing', type: 'rich_text' },
  segmento:             { notionName: 'Segmento',               type: 'select'     },
  metas:                { notionName: 'Metas',                  type: 'rich_text'  }, // join com '\n'
  desafios:             { notionName: 'Desafios',               type: 'rich_text'  }, // join com '\n'
};
```

**Nota importante:** Os nomes de propriedades no Notion variam por database. Se o cliente tiver um database com nomes diferentes, o update vai falhar silenciosamente (Notion ignora propriedades desconhecidas). Tratar erro `APIResponseError` e logar com pino sem conteúdo sensível.

**Casos críticos a testar:**
1. Propriedade já preenchida → `conflicted`, não `updated`
2. Propriedade vazia → `updated` corretamente
3. `forceUpdateAll` → ignora conflito, atualiza tudo
4. Heading "Histórico" existe → entrada inserida corretamente
5. Heading "Histórico" não existe → seção criada antes da inserção
6. JSON inválido do Claude → retorna `{}` sem crashar
7. Notion API error em `updatePageProperties` → logar, não lançar erro (operação não-crítica)
8. `PROPERTY_UPDATE_ALL` com conflitos pendentes → `forceUpdateAll` + clear pending
9. `PROPERTY_UPDATE_ALL` sem conflitos pendentes → mensagem "Não há campos pendentes"

**Nota sobre erros não-críticos:**
O auto-fill de propriedades e registro de histórico são operações de enriquecimento — não devem bloquear a entrega do Plano de Ataque. Se falharem, logar o erro e seguir com a confirmação do plano (sem mencionar o erro ao usuário a não ser que seja relevante).
