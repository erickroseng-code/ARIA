## Story

**As a** usuário,
**I want** gerar relatórios de negócio estruturados com insights via Claude,
**so that** eu possa entender performance e tomar decisões informadas rapidamente.

### Story Details

O relatório consolida dados agregados (Story 5.1) e usa Claude Opus para gerar análises estruturadas. Cada relatório é dividido em 4 seções principais:
- **Executive Summary** (2-3 parágrafos): high-level overview da performance
- **Key Metrics** (5-7 bullet points): números críticos com tendências
- **Insights** (3-5 insights): padrões, gargalos, oportunidades identificadas via IA
- **Recommendations** (3-5 ações): próximos passos práticos para melhoria

O documento é armazenado no Notion em múltiplas páginas (uma por seção), com índice na página raiz. Usuário recebe notificação quando pronto e pode visualizar no Web UI ou Telegram.

## Acceptance Criteria

1. Use aggregated data from Story 5.1 (ReportData model)
2. Claude Opus generates multi-section report: Executive Summary, Key Metrics, Insights, Recommendations
3. Report includes: tasks completed, active clients, meetings conducted, plans created
4. Insights extracted: trends, bottlenecks, opportunities
5. Recommendations actionable: next steps for improving performance
6. Report generated within 2 minutes (Opus latency acceptable for async generation)
7. Report stored in Notion as multi-page document with structured sections
8. User notified when report ready (Telegram + Web UI)
9. Report format: markdown → Notion blocks (sections with metadata)
10. Extensible: add custom report templates in Phase 2

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Claude Opus API timeout (>2 min)** | Report not generated; user waits indefinitely | Medium | Set timeout 2.5 min; queue for retry; show "generating..." status; retry max 2x with exponential backoff |
| **Notion rate limits (3 req/sec)** | Document creation fails; partial pages saved | Low | Batch page creation; respect 3 req/sec limit; queue failed pages for retry; mark as partial |
| **Data validation errors** | Invalid insights; report unusable | Low | Schema validation (Zod); Claude response parsing with fallback; manual review queue |
| **Notification delivery fails** | User not informed report ready | Low | Queue system + retry (3x); fallback to Web UI badge; check on login for missed reports |
| **Report caching conflicts** | Stale data served; user sees outdated insights | Low | Clear cache on data refresh (Story 5.1); TTL 1 hour; manual refresh button in UI |

- Notion: Multi-page document creation via Notion API

- Notification: Telegram + Web UI badge
- Error handling: Queue retry (3x exponential backoff)

### Type GeneratedReport

```typescript
interface GeneratedReport {
  id: string;
  userId: string;
  period: { start: Date; end: Date };
  sections: {
    executiveSummary: string;      // 2-3 paragraphs
    keyMetrics: string[];;          // 5-7 bullet points
    insights: string[];             // 3-5 insights
    recommendations: string[];      // 3-5 actionable items
  };
  notionPageId: string;             // Link to multi-page Notion doc
  generatedAt: Date;
  cacheExpiresAt: Date;
  notificationSentAt?: Date;
  sourceData: {
    clickupMetrics: string;
    notionMetrics: string;
    calendarMetrics: string;
  };
}

interface GeneratedReportRequest {
  userId: string;
  reportData: ReportData;
  regenerate?: boolean;            // Force bypass cache
}
```
