## Story

**As a** usuário,
**I want** usar linguagem natural para criar tarefas sem seguir sintaxe rígida,
**so that** eu possa falar ou digitar comandos naturais como "cria tarefa para Empresa X segunda-feira" e o sistema entenda.

## Acceptance Criteria

1. Claude API extrai entidades estruturadas: título, data, hora, cliente, prioridade, destino (ClickUp/Notion/ambos)
2. Datas relativas interpretadas: "amanhã", "sexta", "próxima semana", "em 2 dias", "14:00"
3. Quando destino não especificado: pergunta "Criar no ClickUp, Notion ou em ambos?"
4. Quando data não especificada: tarefa criada sem prazo com confirmação clara
5. Nome de cliente detectado → tarefa vinculada automaticamente ao cliente (via Notion relation)
6. Quando cliente não encontrado: mostra 3 clientes similares para escolher ou cria como anotação genérica
7. Comandos ambíguos: ARIA pede confirmação antes de criar ("Confirma que é *[titulo]* para *[cliente]* em *[data]*?")
8. Parsing completa em menos de 3 segundos
9. Prioridade extraída de palavras-chave: "urgente", "hoje", "ASAP" → high; "quando tiver tempo" → low
10. Resposta inclui preview: "Vou criar: *[título]* no *[destino]* para *[cliente]* em *[data]* com prioridade *[prioridade]*"

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Claude misinterprets intent** | Wrong task created | Medium | Validation + preview before creation |
| **Date parsing fails** | Task created without date | Low | Default to "no date" with clear feedback |
| **Client matching ambiguous** | Wrong client linked | Low | Show top 3 matches, ask for confirmation |
| **Ambiguous intent not detected** | Task created incorrectly | Low | Request preview confirmation for all tasks |
| **Rate limit cascades** | Multiple APIs hit limit simultaneously | Low | Coordinated rate limiting across all APIs (Whisper, ClickUp, Notion) |

## Out of Scope

- ❌ Multiple tasks in one command ("create 2 tasks")
- ❌ Recurring tasks ("every Monday")
- ❌ Time zone handling (assume local)
- ❌ Custom date formats beyond natural language

### Tipo TaskIntent (Novo)

```typescript
interface TaskIntent {
  title: string;                      // Required
  dueDate?: Date;
  dueDateRelative?: string;           // "amanhã", "sexta", etc
  clientName?: string;                // Company name
  clientId?: string;                  // Notion page ID (if found)
  priority?: 'low' | 'medium' | 'high' | 'urgent';
  destination?: 'clickup' | 'notion' | 'both';
  completeness: 'complete' | 'ambiguous' | 'incomplete';
  ambiguityReason?: string;           // Why it's ambiguous
  clarificationNeeded?: string;       // Question to ask user
}

interface ParseResult {
  intent: TaskIntent;
  confidence: number;                 // 0-1
  preview: string;                    // Human-readable preview
  requiresConfirmation: boolean;
}
```

### Prompt TASK_INTENT_EXTRACTION (Referência)

```typescript
const TASK_INTENT_EXTRACTION_PROMPT = `
Extraia informações de tarefa do texto abaixo.
Retorne JSON válido SEMPRE, mesmo se informação incompleta.

Texto: "${userText}"

JSON obrigatório:
{
  "title": "string ou null",
  "dueDateRelative": "amanhã|sexta|próxima segunda|em 2 dias|14:00|null",
  "clientName": "string ou null",
  "priority": "low|medium|high|urgent ou null",
  "destination": "clickup|notion|both ou null",
  "completeness": "complete|ambiguous|incomplete",
  "ambiguityReason": "string se ambiguous"
}

Regras:
- title: Ação clara (ex: "Ligar para cliente")
- Datas: Sempre relativas quando possível
- Cliente: Nome exato se mencionado
- Prioridade: Extrair de urgência explícita
- completeness: complete = tem título e data; ambiguous = falta contexto; incomplete = falta título
`;
```
