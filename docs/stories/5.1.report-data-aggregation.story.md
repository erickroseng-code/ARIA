## Story

**As a** sistema,
**I want** coletar dados do ClickUp, Notion e Google Calendar em modelo estruturado,
**so that** relatórios podem ser gerados com dados consolidados.

## Acceptance Criteria

1. Coleta do ClickUp: tarefas concluídas, pendentes, atrasadas, criadas no período
2. Coleta do Notion: clientes ativos, Planos criados, reuniões registradas, propriedades auto-fill
3. Coleta do Google Calendar: reuniões realizadas e agendadas
4. Dados normalizados em modelo JSON estruturado (ReportData type)
5. Erro em uma fonte não bloqueia o relatório — marcada como "dados parciais"
6. Modelo extensível para novas fontes
7. Coleta completa em menos de 10 segundos
8. Caching: 1-hour TTL para evitar API calls repetidas
9. Rate limiting: respeita limites de cada API (Notion 3 req/sec, ClickUp 30 req/min)
10. Error handling: timeout per API, fallback to partial data, queue failed requests

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **API rate limits** | Aggregation fails | Medium | Cache results for 1h; batch requests; queue system |
| **One source fails** | Incomplete report | Low | Graceful degradation; mark as partial data |
| **Slow aggregation** | Exceeds timeout | Low | Parallel fetching; set timeout for each API (3s per API) |
| **Data normalization bugs** | Wrong report data | Low | Unit tests for each data source; validation schemas |

## Out of Scope

- ❌ Historical data (current period only)
- ❌ Custom date range filtering (configurable in config only)
- ❌ Export to file (Phase 2)

## Dev Notes

### Dependencies
- Depends on Story 3.2: ClickUp task data
- Depends on Story 3.3: Notion task data
- Depends on Story 4.1: Google Calendar events
- Used by Story 5.2: Report generation uses aggregated data

### Architecture Reference
- Service: ReportDataAggregationService (orchestrator)
- Caching: 1-hour TTL (configurable)
- Rate limiting: respect each API's limits (Notion 3 req/sec, ClickUp 30 req/min)
- Parallel fetching: Promise.all with individual timeouts (3s per API)
- Fallback: mark as "partial data" if one source fails

### Tipo ReportData

```typescript
interface ReportData {
  period: { start: Date; end: Date };
  clickup: {
    tasksCompleted: number;
    tasksPending: number;
    tasksOverdue: number;
    tasksCreated: number;
  };
  notion: {
    activeClients: number;
    plansCreated: number;
    meetingsRecorded: number;
    propertiesFilled: number;
    propertyConflicts: number;
  };
  calendar: {
    meetingsScheduled: number;
    meetingsCompleted: number;
    hoursInMeetings: number;
  };
  generatedAt: Date;
  cacheExpiresAt: Date;
  isPartialData: boolean;  // if one source failed
  errors?: string[];       // error messages from each source
}
```
