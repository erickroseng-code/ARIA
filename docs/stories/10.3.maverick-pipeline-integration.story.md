# Story 10.3: Maverick — Integração do Carrossel no Pipeline e UI

## Status
Ready for Dev

## Story

**As a** usuário do Maverick na interface web,
**I want** ver os slides do carrossel gerados automaticamente após os scripts,
**so that** eu tenha a estrutura completa pronta para criar o design sem precisar chamar nenhum endpoint separado.

## Acceptance Criteria

1. **Geração automática:** Após receber os scripts via SSE, o frontend chama automaticamente `POST /api/maverick/carousel` para cada script e exibe os slides.
2. **UI dos slides:** No `MaverickSession.tsx`, após a seção de scripts, nova seção "🎨 Estrutura do Carrossel" mostra os slides em cards horizontais (scroll). Cada card mostra: número, type badge, título, body truncado, visual_hint.
3. **Estado de loading:** Enquanto o carrossel está sendo gerado, exibe "🎨 Gerando estrutura do carrossel..." com spinner.
4. **Botão de exportar HTML:** Botão "⬇️ Exportar HTML" baixa o `htmlExport` como arquivo `.html` para o usuário.
5. **Opcional Figma:** Se `figmaUrl` estiver presente na resposta, exibe botão "Abrir no Figma" com link externo.
6. **Feature flag no frontend:** Se `MAVERICK_CAROUSEL_ENABLED !== 'true'` (verificado via endpoint), a seção de carrossel é simplesmente omitida — sem erro visível.
7. **Um carrossel por script:** Se houver múltiplos scripts, gera carrossel para o script selecionado/em foco.

## Scope

**IN:**
- Modificar `MaverickSession.tsx` — nova seção de carrossel após scripts
- Chamar `POST /api/maverick/carousel` após scripts carregados
- Componente de slide card (inline em MaverickSession.tsx, sem criar novo arquivo)
- Botão de download HTML

**OUT:**
- Não salvar carrossel no banco de dados (histórico de scripts já existe)
- Não criar novo componente separado (manter no MaverickSession.tsx)
- Não integrar com Figma além do link externo

- [ ] **Task 2** — Função `generateCarousel(script)` que chama o endpoint (AC: 1)
  ```ts
  const generateCarousel = useCallback(async (script: ScriptData) => {
    setCarouselState(s => ({ ...s, loading: true, error: null }));
    try {
      const res = await fetch(`${API_URL}/api/maverick/carousel`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ script }),
      });
      if (!res.ok) throw new Error('Falha ao gerar carrossel');
      const data = await res.json();
      setCarouselState({ loading: false, carousel: data.carousel, htmlExport: data.htmlExport, figmaUrl: data.figmaUrl ?? null, error: null });
    } catch (err: any) {
      setCarouselState(s => ({ ...s, loading: false, error: err.message }));
    }
  }, []);
  ```
  - Chamar `generateCarousel(scripts[0])` automaticamente quando scripts chegarem via SSE

- [ ] **Task 4** — Função `downloadHtml()` (AC: 4)
  ```ts
  function downloadHtml(html: string) {
    const blob = new Blob([html], { type: 'text/html' });
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = `maverick-carousel-${Date.now()}.html`;
    a.click();
    URL.revokeObjectURL(url);
  }
  ```

## Dev Notes

### Disparar geração automática

Após o evento SSE `scripts`, quando `parsedScripts` for atualizado, usar `useEffect` para disparar automaticamente:

```ts
useEffect(() => {
  if (scripts.length > 0 && !carouselState.carousel && !carouselState.loading) {
    generateCarousel(scripts[0]); // gera para o primeiro script
  }
}, [scripts]);
```

### Tipo CarouselStructure no frontend

Definir interface localmente em `MaverickSession.tsx` (não importar do backend):
```ts
interface Slide { position: number; type: 'cover'|'content'|'cta'; title: string; body: string; visual_hint: string; }
interface CarouselStructure { title: string; format: string; total_slides: number; slides: Slide[]; }
```
