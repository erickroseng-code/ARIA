# Story 12.2: Carousel Screenshot Pipeline (Playwright)

## Status
Done

## Story

**As a** sistema de automação,
**I want** que cada slide HTML do carrossel seja capturado como imagem PNG 1080×1080px via Playwright,
**so that** os arquivos possam ser baixados e publicados diretamente no Instagram sem redimensionamento.

## Acceptance Criteria

1. **Função `screenshotSlide(html: string, outputPath: string): Promise<void>`** implementada em `carousel-designer/screenshot.ts`.
2. **Dimensões exatas 1080×1080px** (formato Instagram square).
3. **Playwright headless** com configurações de compatibilidade para ambientes sem GPU: `--no-sandbox`, `--disable-gpu`, `--disable-dev-shm-usage`.
4. **Integração no batch:** `POST /api/maverick/weekly-batch` chama `screenshotSlide` para cada slide e salva em `/tmp/batch-{batchId}/topico-{n}/slide-{n}.png`.
5. **Cleanup automático:** arquivos em `/tmp/batch-{batchId}/` são deletados após 24h.
6. **Fallback gracioso:** se Playwright falhar (sem Chrome instalado), o batch continua retornando apenas HTML (sem PNGs), com `screenshots: false` no response.
7. **Instalação do browser** verificada no startup: se não instalado, logar warning mas não falhar o servidor.

  ```ts
  // aria/apps/api/src/modules/maverick/carousel-designer/screenshot.ts

  import * as fs from 'fs';
  import * as path from 'path';

  let chromium: any;

  async function getBrowser() {
    if (!chromium) {
      try {
        const { chromium: cr } = await import('playwright');
        chromium = cr;
      } catch {
        throw new Error('Playwright não instalado. Execute: npx playwright install chromium');
      }
    }
    return chromium;
  }

  export async function screenshotSlide(html: string, outputPath: string): Promise<void> {
    const cr = await getBrowser();
    const browser = await cr.launch({
      args: ['--no-sandbox', '--disable-gpu', '--disable-dev-shm-usage'],
      headless: true,
    });

    try {
      const page = await browser.newPage();
      await page.setViewportSize({ width: 1080, height: 1080 });
      await page.setContent(html, { waitUntil: 'networkidle' });

      // Aguarda fontes carregarem
      await page.waitForTimeout(500);

      const dir = path.dirname(outputPath);
      fs.mkdirSync(dir, { recursive: true });

      await page.screenshot({ path: outputPath, type: 'png', clip: { x: 0, y: 0, width: 1080, height: 1080 } });
    } finally {
      await browser.close();
    }
  }

  export async function screenshotBatch(
    slides: Array<{ html: string }>,
    batchDir: string
  ): Promise<string[]> {
    const paths: string[] = [];
    for (let i = 0; i < slides.length; i++) {
      const outputPath = path.join(batchDir, `slide-${String(i + 1).padStart(2, '0')}.png`);
      await screenshotSlide(slides[i].html, outputPath);
      paths.push(outputPath);
    }
    return paths;
  }
  ```

## Dev Notes

### HTML dos slides vs HTML do carousel completo
A função `generateCarouselHtml` retorna o HTML do carrossel inteiro (todos os slides em uma página). Para screenshots individuais por slide, precisamos de uma das opções:

**Opção A (recomendada):** Modificar `generateCarouselHtml` para também exportar `generateSlideHtml(slide, theme)` que retorna HTML de um slide individual (página completa 1080×1080).

**Opção B:** Usar seletores CSS no Playwright para capturar cada `.slide` individualmente do HTML completo.

Implementar **Opção A** — criar `generateSlideHtml()` em `html-export.ts` que encapsula um único slide em página standalone 1080×1080.

### Playwright no Render (Free Tier)
- Memory limit: 512MB no free tier
- Chromium headless: ~150-200MB por instância
- Estratégia: uma instância por slide (não reutilizar browser entre slides) para evitar memory leaks
- Se free tier não suportar, considerar upgrade para Render Starter ($7/mês, 1GB RAM)

### Alternativa sem Playwright
Se Playwright for inviável no Render free tier, usar `puppeteer-core` + `@sparticuz/chromium` (otimizado para Lambda/serverless com ~50MB).
