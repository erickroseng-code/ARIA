# Story 12.4: GitHub Actions Cron + Telegram Notification

## Status
Done

## Story

**As a** criador de conteúdo,
**I want** receber automaticamente toda segunda-feira às 8h (BRT) os carrosséis da semana no Telegram,
**so that** nunca precise lembrar de gerar conteúdo manualmente — a Aria entrega sozinha.

## Acceptance Criteria

1. **`.github/workflows/maverick-weekly.yml`** criado com trigger cron `0 11 * * 1` (segunda às 11h UTC = 8h BRT).
2. **Workflow executa:**
   1. `POST $ARIA_API_URL/api/maverick/weekly-batch` com tópicos predefinidos
   2. Extrai `batchId` da resposta
   3. `POST $ARIA_API_URL/api/maverick/weekly-batch/{batchId}/send-telegram`
3. **Tópicos predefinidos** configurados via GitHub Actions secret `MAVERICK_WEEKLY_TOPICS` (JSON array de strings).
4. **Timeout de 10 minutos** no step do weekly-batch (pipeline pode demorar).
5. **Notificação de falha:** se o workflow falhar, GitHub Actions envia notificação de erro ao Telegram via step de fallback.
6. **Workflow também pode ser disparado manualmente** (`workflow_dispatch` com input opcional de tópicos).
7. **Logs limpos:** secrets não aparecem em output do workflow.

## Scope

**IN:**
- Criar `.github/workflows/maverick-weekly.yml`
- Documentar como configurar os secrets necessários
- Adicionar GitHub Actions secrets `MAVERICK_WEEKLY_TOPICS` e `ARIA_API_URL` (se não existem)

**OUT:**
- Não modificar código da API (stories 12.1-12.3 já cobrem isso)
- Não criar workflow para outros dias da semana
- Não configurar retry automático (GitHub Actions retentar é manual)

## Tasks / Subtasks

- [x] **Task 1** — Criar `.github/workflows/maverick-weekly.yml` (AC: 1, 2, 3, 4, 5, 6, 7)
  ```yaml
  name: Maverick Weekly — Carrosséis Automáticos

  on:
    schedule:
      - cron: '0 11 * * 1'  # Segunda-feira às 8h BRT (UTC-3 = 11h UTC)
    workflow_dispatch:
      inputs:
        topics:
          description: 'Tópicos JSON (ex: ["emagrecimento", "mindset"])'
          required: false
          type: string

  jobs:
    generate-carousels:
      runs-on: ubuntu-latest
      timeout-minutes: 15

      steps:
        - name: Generate weekly batch
          id: batch
          run: |
            # Usa tópicos do input manual se fornecidos, senão usa os configurados
            TOPICS='${{ github.event.inputs.topics }}'
            if [ -z "$TOPICS" ]; then
              TOPICS='${{ secrets.MAVERICK_WEEKLY_TOPICS }}'
            fi

            echo "🚀 Gerando carrosséis para tópicos: $TOPICS"

            RESPONSE=$(curl \
              --silent \
              --max-time 600 \
              --fail \
              --header "Content-Type: application/json" \
              --data "{\"topics\": $TOPICS, \"theme\": \"dark\"}" \
              "${{ secrets.ARIA_API_URL }}/api/maverick/weekly-batch")

            BATCH_ID=$(echo "$RESPONSE" | python3 -c "import sys,json; print(json.load(sys.stdin)['batchId'])")
            echo "batch_id=$BATCH_ID" >> $GITHUB_OUTPUT
            echo "✅ Batch gerado: $BATCH_ID"

        - name: Send to Telegram
          run: |
            echo "📤 Enviando carrosséis para Telegram..."

            curl \
              --silent \
              --max-time 120 \
              --fail \
              --request POST \
              "${{ secrets.ARIA_API_URL }}/api/maverick/weekly-batch/${{ steps.batch.outputs.batch_id }}/send-telegram"

            echo "✅ Carrosséis enviados ao Telegram!"

        - name: Notify failure
          if: failure()
          run: |
            curl --silent \
              --request POST \
              --header "Content-Type: application/json" \
              --data "{
                \"chat_id\": \"${{ secrets.TELEGRAM_CHAT_ID }}\",
                \"text\": \"❌ *Maverick Weekly falhou!*\n\nVerifique os logs: https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }}\",
                \"parse_mode\": \"Markdown\"
              }" \
              "https://api.telegram.org/bot${{ secrets.TELEGRAM_BOT_TOKEN }}/sendMessage"
  ```

## Dev Notes

### Horário do Cron
- `0 11 * * 1` = toda segunda-feira às 11:00 UTC
- BRT = UTC-3, então 11h UTC = 8h BRT ✅
- Durante horário de verão (BRST = UTC-2): `0 10 * * 1` para manter 8h BRT

### Timeout do curl
- `--max-time 600` = 10 minutos para o weekly-batch
- O pipeline completo (3 tópicos × scraping + LLM + Playwright) pode levar 5-8 min
- GitHub Actions timeout é 15 min (`timeout-minutes: 15`)

### Render Cold Start
Se a API no Render entrar em spin-down (UptimeRobot pode falhar ocasionalmente), o primeiro curl pode demorar 30s a mais. O `--max-time 600` é suficiente para absorver isso.

### Secrets no Output
- Nunca usar `echo ${{ secrets.X }}` diretamente
- Usar `>> $GITHUB_OUTPUT` apenas para valores não-sensíveis (batchId)
- Python3 para parse JSON é mais seguro que `jq` (não requer instalação no ubuntu-latest)

### Alternativa: jq
Se preferir `jq` (disponível no ubuntu-latest por padrão):
```bash
BATCH_ID=$(echo "$RESPONSE" | jq -r '.batchId')
```
