# Story 11.1: Dockerization da API Aria

## Status
InProgress

## Story

**As a** operador do sistema Aria,
**I want** que a API possa ser executada em container Docker,
**so that** seja possível fazer deploy no Render e em qualquer ambiente cloud com configuração mínima.

## Acceptance Criteria

1. **`aria/apps/api/Dockerfile`** multi-stage criado: stage `build` (compila TypeScript) + stage `runtime` (Node.js slim, apenas `dist/`).
2. **`aria/apps/api/.dockerignore`** exclui: `node_modules`, `.env`, `*.log`, `dist`, `dev.db`, `prisma/*.db`.
3. **`aria/apps/api/.env.example`** atualizado com TODOS os secrets necessários para produção, com valores placeholder e comentário explicativo para cada um.
4. **Build local funciona:** `docker build -t aria-api -f aria/apps/api/Dockerfile aria/apps/api` conclui sem erros.
5. **Container inicia:** `docker run --env-file aria/apps/api/.env aria-api` inicia o servidor e `/health` retorna 200.
6. **Zero secrets** expostos no `Dockerfile` (nenhum `ENV SECRET=valor_real`).
7. **Imagem final < 500MB** (Node.js Alpine ou Slim como base no runtime stage).

## Scope

**IN:**
- Criar `aria/apps/api/Dockerfile`
- Criar `aria/apps/api/.dockerignore`
- Atualizar `aria/apps/api/.env.example` com todos os secrets
- Verificar que `package.json` tem `"start": "node dist/apps/api/src/server.js"` (já existe)

**OUT:**
- Não fazer push para Docker Hub
- Não configurar Render ainda (Story 11.2)
- Não alterar código da API

## Dependencies

- `aria/apps/api/package.json` já tem script `start` configurado ✅
- `aria/apps/api/src/server.ts` como entrypoint

## Complexity Estimate

**T-Shirt:** Small (S)
**Story Points:** 3
**Dev Time:** ~3 horas

## Business Value

Container Docker é o prerequisito técnico para qualquer deploy em cloud (Render, Railway, Fly.io). Sem ele, a Aria não pode operar 24/7 de forma autônoma sem infraestrutura local.

  ```dockerfile
  # ── Stage 1: Build ──────────────────────────────────────────
  FROM node:20-alpine AS build
  WORKDIR /app

  # Instala dependências primeiro (cache layer)
  COPY package.json package-lock.json ./
  RUN npm ci --include=dev

  # Copia código e compila
  COPY . .
  RUN npm run build

  # Remove devDependencies
  RUN npm prune --production

  # ── Stage 2: Runtime ─────────────────────────────────────────
  FROM node:20-alpine AS runtime
  WORKDIR /app

  ENV NODE_ENV=production

  # Copia apenas o necessário
  COPY --from=build /app/dist ./dist
  COPY --from=build /app/node_modules ./node_modules
  COPY --from=build /app/package.json ./package.json
  COPY --from=build /app/prisma ./prisma

  # Cria diretório para banco SQLite em produção
  RUN mkdir -p /app/data

  EXPOSE 3001

  CMD ["node", "dist/apps/api/src/server.js"]
  ```

## Dev Notes

### Entrypoint atual
`package.json` → `"start": "node dist/apps/api/src/server.js"`

O path de build do TypeScript deve ser verificado no `tsconfig.json` — o `outDir` define onde o `dist/` é gerado.

### Prisma em produção
O banco SQLite em produção deve ser montado como volume no Render. O `DATABASE_URL` em produção deve apontar para `/app/data/prod.db` (ou Turso/libsql para persistência real).

### Health check endpoint
Verificar se `/health` já existe no server. Se não, adicionar antes do deploy (Story 11.2 inclui isso).

## Dev Notes (implementação)

### Decisão: contexto de build = `aria/` (monorepo root)
O Dockerfile usa a pasta `aria/` como contexto (não `aria/apps/api/`) porque a API depende de `@aria/core` e `@aria/integrations` (workspace packages). O comando correto de build é:
```bash
docker build -t aria-api -f aria/apps/api/Dockerfile aria/
```

### Arquivos criados/modificados
- `aria/apps/api/Dockerfile` — multi-stage build (build + runtime)
- `aria/apps/api/.dockerignore` — regras locais da API
- `aria/.dockerignore` — regras do monorepo (contexto correto)
- `aria/.env.example` — atualizado com 35+ variáveis documentadas

### Task 4 (pendente verificação manual)
Docker Desktop não estava acessível no ambiente de desenvolvimento. Executar manualmente:
```bash
docker build -t aria-api -f aria/apps/api/Dockerfile aria/
docker run --env-file aria/.env -p 3001:3001 aria-api
curl http://localhost:3001/health
```
