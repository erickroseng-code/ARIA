## Story

**As a** usuário,
**I want** enviar mensagens de voz via Telegram ou Web UI e ter o áudio transcrito automaticamente,
**so that** eu possa criar tarefas por voz sem precisar digitar.

## Acceptance Criteria

1. Sistema aceita áudio via Telegram (voice message nativa) e Web UI (botão de gravação)
2. Áudio enviado como buffer local (base64) com suporte a WAV/MP3/OGG (até 60 segundos)
3. Whisper API transcorre áudio em menos de 5 segundos para arquivo <10MB
4. Transcrição retorna como texto UTF-8, sem pontuação automática (usuario adiciona depois)
5. Arquivo de áudio deletado imediatamente após transcrição (NFR11)
6. Telegram: mensagem "🎙 Transcrevendo áudio..." enquanto processa
7. Web UI: spinner de loading + progresso de transcrição
8. Erro em transcrição: retry 2x com exponential backoff (1s, 2s), depois falha com mensagem clara
9. Transcrição vazia: alerta "Áudio inválido ou sem conteúdo"
10. Suporte a áudios de 1-60 segundos (fora desse range: reject com mensagem)

## Risks & Mitigation

| Risk | Impact | Likelihood | Mitigation |
|------|--------|-----------|-----------|
| **Whisper API timeout** | User cannot transcribe | Medium | Implement 30s timeout + 2x retry with backoff |
| **Audio format incompatibility** | Bad transcription or error | Low | Normalize to WAV on client before sending |
| **Memory leak (large audio)** | OOM crash after 10+ transcriptions | Low | Stream buffer, not concat; delete immediately |
| **Rate limit hit** | Requests fail | Low | Queue system (Story 3.4 handles global rate limiting) |
