## Story

**As a** usuário,
**I want** que a ARIA interprete os documentos enviados e gere resumos estratégicos e checklists automaticamente,
**so that** eu receba conteúdo estruturado e pronto para inserção no Plano de Ataque do cliente no Notion, sem precisar ler e resumir os documentos manualmente.

## Acceptance Criteria

1. Conteúdo extraído dos documentos (Story 2.2) enviado ao `claude-sonnet-4-6` com prompt estruturado por tipo de setor
2. Geração de **Resumo do Setor Comercial** — pontos-chave, objetivos, desafios identificados
3. Geração de **Resumo do Setor de Marketing** — estratégias, oportunidades, posicionamento
4. Geração de **Análise Integrada** — cruzamento entre setores, gaps, sinergias e recomendações
5. Geração de **Checklists Práticos** — ações concretas ordenadas por prioridade (alta/média/baixa)
6. Output com **formatação Markdown rica** (FR19): títulos `#`, listas `-`, tabelas, callouts `>`, separadores `---`
7. **Preview** do conteúdo gerado enviado ao usuário via streaming antes de confirmar criação no Notion
8. Trigger ativado quando usuário digita "pronto" (ou variações) após envio de documentos na sessão
9. Sistema detecta automaticamente o tipo de setor pelo conteúdo (comercial, marketing, RH, etc.) quando não especificado
10. Suporte a múltiplos documentos: cada documento gera sua seção; Análise Integrada consolida todos

| **Streaming chunks fragmented** | Frontend parsing issues | Low | Use delimited JSON objects in stream |

export function buildUserMessage(docs: ProcessedDocument[], clientName: string): string {
  const docSections = docs.map((doc, i) =>
    `## DOCUMENTO ${i + 1}: ${doc.originalName}\n\n${doc.extractedText}`
  ).join('\n\n---\n\n');

  return `Cliente: ${clientName}\n\nDocumentos para análise:\n\n${docSections}`;
}

### DocumentAnalysisService — Referência

```typescript
// packages/core/src/documents/DocumentAnalysisService.ts
import Anthropic from '@anthropic-ai/sdk';
import { ANALYSIS_SYSTEM_PROMPT, buildUserMessage } from './prompts/analysis.prompt';
import type { ProcessedDocument, GeneratedAnalysis } from '@aria/shared';

export class DocumentAnalysisService {
  constructor(private claude: Anthropic) {}

  async *analyzeDocuments(
    docs: ProcessedDocument[],
    clientName: string
  ): AsyncGenerator<string> {
    if (docs.length === 0) {
      throw new AppError('No documents to analyze', 'DOC_005', { statusCode: 400 });
    }

    const stream = this.claude.messages.stream({
      model: 'claude-sonnet-4-6',
      max_tokens: 8192,
      system: ANALYSIS_SYSTEM_PROMPT,
      messages: [
        { role: 'user', content: buildUserMessage(docs, clientName) }
      ],
    });

    for await (const chunk of stream) {
      if (chunk.type === 'content_block_delta' && chunk.delta.type === 'text_delta') {
        yield chunk.delta.text;
      }
    }
  }

  async buildAnalysis(
    docs: ProcessedDocument[],
    clientName: string
  ): Promise<string> {
    let fullText = '';
    for await (const chunk of this.analyzeDocuments(docs, clientName)) {
      fullText += chunk;
    }
    return fullText;
  }

  async detectSectorType(text: string): Promise<string> {
    // Classificação rápida com haiku
    const result = await this.claude.messages.create({
      model: 'claude-haiku-4-5-20251001',
      max_tokens: 50,
      system: 'Responda em uma palavra: qual setor empresarial este texto representa? Ex: comercial, marketing, rh, financeiro, operacoes, ti, juridico',
      messages: [{ role: 'user', content: text.slice(0, 500) }],
    });
    return result.content[0].type === 'text'
      ? result.content[0].text.trim().toLowerCase()
      : 'geral';
  }
}
```

### Limite de Tokens — Estimativa

| Cenário | Tokens estimados |
|---------|-----------------|
| 1 documento (5 páginas) | ~3.000 input + ~2.000 output |
| 3 documentos (5 pág cada) | ~9.000 input + ~4.000 output |
| 5 documentos (10 pág cada) | ~20.000 input + ~6.000 output |

`claude-sonnet-4-6` suporta até 200K tokens de contexto — margem suficiente para o MVP.
`max_tokens: 8192` para output é adequado para análises detalhadas.

### Estratégia Telegram — Mensagens Longas

O Telegram tem limite de 4096 caracteres por mensagem. Para análises longas:

```typescript
function splitIntoTelegramMessages(text: string, maxLength = 3900): string[] {
  const sections = text.split(/(?=\n## |\n### )/);
  const messages: string[] = [];
  let current = '';

  for (const section of sections) {
    if ((current + section).length > maxLength) {
      if (current) messages.push(current.trim());
      current = section;
    } else {
      current += section;
    }
  }
  if (current) messages.push(current.trim());
  return messages;
}
```

### FR19 — Formatação Rica

Instrução explícita no system prompt garante que Claude usa:
- `##` / `###` para hierarquia
- `-` para listas
- `**negrito**` para destaques
- `>` para callouts estratégicos
- `---` para separadores de seção
- `- [ ]` para checklist de ações
